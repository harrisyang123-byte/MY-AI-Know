# 启动项目

你收到了"启动项目"指令。**不要直接开始做任何具体任务。**

## 第一步：加载项目导航系统

按优先级尝试读取以下文件（读到哪个用哪个）：

1. `atoms/01.CORE/agents/project-guide.md` — 项目总指南（最优先）
2. `README.md` — 项目说明
3. `CLAUDE.md` — 已自动加载，了解当前规范

## 第二步：报告当前状态

向用户汇报：
- 你读到了什么文件
- 项目当前的进展状态（哪些场景已完成、哪些进行中、下一步建议）
- **问用户：你想从哪里开始？**

## 第三步：等待用户指示

**禁止自行决定下一步做什么。** 可能的用户意图：
- "继续 S04 分镜" → 加载 `/分镜` 工作流
- "生成提示词" → 切换到提示词生成模式
- "帮我看看代码" → 进入探索阶段
- 其他任何需求 → 按创意协作三阶段工作流处理

## 核心原则
> **启动 = 加载导航，不是替用户做决定。**
> 项目有自己的流程体系（atoms/），让项目告诉你该走哪条路，而不是你猜。
