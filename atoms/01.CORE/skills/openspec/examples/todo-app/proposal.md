# 提案：给 Todo 应用添加优先级功能

## 📋 基本信息
- **变更名称**: add-task-priority
- **创建日期**: {{date}}
- **创建者**: {{user}}
- **状态**: draft

## 🎯 目标
为 Todo 应用添加任务优先级功能，支持高、中、低三级优先级，帮助用户更好地管理任务重要性。

## 📊 业务价值
- 用户可以根据任务重要性进行排序和筛选
- 提高任务管理效率
- 更直观的可视化展示

## 🎨 功能范围

### ✅ 包含的功能
1. **数据模型扩展**
   - Task 模型添加 priority 字段
   - 支持优先级筛选和排序

2. **API 端点扩展**
   - 创建任务时指定优先级
   - 按优先级筛选任务
   - 更新任务优先级

3. **用户界面**
   - 任务列表显示优先级标签
   - 优先级筛选控件
   - 优先级选择器（创建/编辑任务时）

4. **业务逻辑**
   - 默认优先级（中）
   - 优先级验证规则

### ❌ 不包含的功能
  - 子任务优先级继承
  - 自动优先级分配
  - 优先级历史记录
  - 多用户优先级协同

## 🧪 验收标准

### 数据层
- [ ] Task 模型有 priority 字段（enum: 'high' | 'medium' | 'low'）
- [ ] 数据库迁移脚本正确添加列
- [ ] 默认值设置为 'medium'

### API 层
- [ ] `GET /tasks` 支持 priority 查询参数
- [ ] `POST /tasks` 接受 priority 字段
- [ ] `PATCH /tasks/:id` 可更新 priority 字段
- [ ] API 文档已更新

### 业务逻辑
- [ ] 优先级验证：必须是 'high'、'medium'、'low' 之一
- [ ] 默认优先级为 'medium'
- [ ] 按优先级排序：high → medium → low

### 用户界面
- [ ] 任务列表项显示优先级标签
- [ ] 标签颜色：high=红色、medium=橙色、low=绿色
- [ ] 筛选控件：全部/高/中/低
- [ ] 创建/编辑表单包含优先级选择器

### 测试
- [ ] 单元测试覆盖优先级逻辑
- [ ] 集成测试验证 API 行为
- [ ] UI 组件测试
- [ ] 测试覆盖率 ≥ 80%
