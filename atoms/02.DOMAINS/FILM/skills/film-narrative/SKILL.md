# Film Narrative

两种工作模式，开始前先确认用户需要哪种：

- **模式 A：从零创作** — 从主题/想法出发，构建完整叙事骨架
- **模式 B：剧本优化** — 输入已有剧本/大纲，诊断结构问题，给出改写方案

方法论基础（三层分工）：
- **Egri**（《编剧艺术》）：前提驱动 + 人物三维骨架 → 管"为什么讲"
- **Snyder**（《救猫咪》）：Logline + 类型定位 + 15节拍 + 写作规则 → 管"讲什么、讲多快"
- **McKee**（《故事》）：场景价值反转 + Gap 理论 → 管"每场戏怎么讲"

## 模式 A：从零创作

### 第一步：确立前提（Egri）

前提是整个故事的论点——一句话证明某个关于人性的命题。

**前提公式**：`[人物特质/行为] + [冲突] → [结果]`

示例：「盲目的野心导致自我毁灭」（麦克白）/ 「牺牲性的爱征服绝望」

前提必须同时包含：人物暗示 + 冲突暗示 + 结局暗示。不是主题词，不是情节摘要，是可以被故事证明的命题。

### 第二步：写出 Logline（Snyder）

Logline 是剧本的"宪法"——先有 logline，再写剧本。Snyder 认为 logline 应该先于一切，是整个创作过程的锚点。

**Logline 四要素检验**：

1. **讽刺性（Irony）**：好的 logline 内含反差或悖论，让人一听就想知道"然后呢"
   - 示例：「一个恐惧大海的警长，必须独自出海猎杀一头巨鲨」（大白鲨）
   - 反面：「一个警长追捕一条鲨鱼」（无讽刺性，无张力）

2. **视觉画面感（Compelling Mental Picture）**：读完 logline，脑海中能立刻浮现出具体场景和可能的高潮戏

3. **受众与成本暗示（Audience and Cost）**：logline 应该隐含这是什么类型的观众、大概是什么量级的制作

4. **杀手级标题（Killer Title）**：标题和 logline 是"一二连击"，标题要"说清楚是什么"又有记忆点

**Logline 模板**（Snyder 版）：

当 [催化事件] 发生后，[有缺陷的主角] 必须 [外部目标]，
但 [对立力量] 阻止他，而他真正需要学会的是 [内心需求]。

### 第三步：确定类型（Snyder 10种类型）

Snyder 的类型不是传统的动作/爱情/恐怖，而是基于叙事结构的分类。确定类型后，节拍表的填写方式会有所不同。

**10种类型快速定位**：

| 类型 | 核心公式 | 典型例子 |
|------|---------|---------|
| 怪物入侵 Monster in the House | 封闭空间 + 怪物 + 有人犯了罪引来怪物 | 大白鲨、异形、驱魔人 |
| 寻宝之旅 Golden Fleece | 主角踏上旅程寻找某物，最终发现自己 | 星球大战、绿野仙踪、回到未来 |
| 愿望成真 Out of the Bottle | 愿望被实现，但代价是学会珍惜原本的生活 | 大话王、变身国王 |
| 普通人遇上大麻烦 Dude with a Problem | 普通人被卷入非凡处境 | 虎胆龙威、泰坦尼克号、辛德勒的名单 |
| 成长仪式 Rite of Passage | 人生转折点，主角必须接受某个痛苦的真相 | 普通人、克莱默夫妇、毕业生 |
| 伙伴情谊 Buddy Love | 两个不同的人相遇，互相改变，爱情或友情 | 阿甘正传、雨人、诺丁山 |
| 为何如此 Whydunit | 不是"谁干的"而是"为什么"，揭示人性黑暗 | 唐人街、JFK、内幕 |
| 傻瓜得胜 Fool Triumphant | 被低估的局外人对抗体制，最终证明自己 | 阿甘正传、查理和巧克力工厂、莫扎特传 |
| 体制内外 Institutionalized | 个人 vs 群体/机构，关于归属与身份的代价 | 飞越疯人院、教父、动物屋 |
| 超级英雄 Superhero | 非凡的人在平凡世界中的孤独与代价 | 超人、蝙蝠侠、角斗士、美丽心灵 |

详细类型定义和节拍差异见 [references/snyder-genres.md](references/snyder-genres.md)。

### 第四步：建立人物三维骨架（Egri）

每个核心角色输出三个维度——缺少任何一维，人物就是平的：

姓名：
一句话定位：

生理维度：外貌、健康、身体能力 → 影响他如何被世界对待

社会维度：阶层、职业、家庭、教育 → 影响他如何看待世界

心理维度：
  核心欲望 Want：表面上想要什么
  核心需求 Need：真正需要什么（通常与欲望冲突）
  最大恐惧：最不愿面对的真相
  核心矛盾：身上最根本的内在对立

弧光起点：故事开始时相信什么是真的
弧光终点：故事结束时学到了什么（或拒绝学到什么）

**对立统一**（Egri 核心）：主角与对手必须互相需要、互相定义——缺少任何一方，冲突就无法成立。

**Save the Cat 时刻**（Snyder）：在铺垫阶段，主角必须做一件让观众与他同步的事。不一定是字面上"救猫"，可以是展示他的幽默、善良、或让观众理解他的处境。目的是让观众在故事真正开始前就站在主角一边。

### 第五步：填写节拍表（Snyder）

按 15 个节拍顺序输出，每个节拍包含：节拍名 | 位置（页/百分比）| 事件（一句话）| 价值状态（+/-）| 情绪目标

**15节拍精确位置**（110页剧本参考）：

| 节拍 | 位置 | 核心功能 |
|------|------|---------|
| 1 开场画面 | p.1 / 0-1% | 主角"之前"状态的视觉快照，与终幕画面形成对照 |
| 2 主题陈述 | p.5 / ~5% | 有人（通常不是主角）说出故事真正要讲的道理 |
| 3 铺垫 | p.1-10 / 1-10% | 展示主角日常、缺陷、人际关系；必须包含 Save the Cat 时刻 |
| 4 催化剂 | p.12 / ~10% | 打破日常的事件，故事真正开始；必须在第12页前发生 |
| 5 争论 | p.12-25 / 10-20% | 主角犹豫，内心挣扎；主角必须主动选择，不能被动漂入第二幕 |
| 6 进入第二幕 | p.25 / 20% | 主角做出选择，主动进入新世界，不能回头 |
| 7 B故事 | p.30 / ~22% | 次要线索启动，通常是承载主题的关系线；B故事在第三幕提供答案 |
| 8 娱乐与游戏 | p.30-55 / 20-50% | 兑现故事的核心卖点（promise of the premise），探索新世界 |
| 9 中点 | p.55 / 50% | 表面胜利（假胜利）或表面失败（假失败），风险升级 |
| 10 坏人逼近 | p.55-75 / 50-75% | 外部压力增大，内部团队开始瓦解，主角缺陷开始反噬 |
| 11 一败涂地 | p.75 / ~75% | 主角失去一切，比开始时更糟，通常伴随"死亡气息"（象征或真实） |
| 12 灵魂黑夜 | p.75-85 / 75-80% | 主角在绝望中反思；B故事的答案在这里开始浮现 |
| 13 进入第三幕 | p.85 / ~80% | 顿悟，通常来自B故事，决定最后一搏 |
| 14 结局 | p.85-110 / 80-99% | 用新学到的东西解决问题；坏人按层级被消灭（小喽啰先，Boss最后） |
| 15 终幕画面 | p.110 / 99-100% | 与开场画面呼应，是主题改变的视觉证明 |

详细节拍定义见 [references/save-the-cat-beats.md](references/save-the-cat-beats.md)。

**McKee 价值状态检验**：每个节拍标注此刻主角核心价值的正负极（+/-）。连续多个节拍价值状态不变 → 中段塌陷预警。

### 第六步：场景卡片（Snyder）

在节拍表和场景清单之间，用场景卡片系统检验整体节奏。

将故事拆成约 40 张场景卡片，分布在四行：
- 第一行：第一幕（p.1-25）
- 第二行：第二幕前半（p.25-55，到中点）
- 第三行：第二幕后半（p.55-85，到第三幕转折）
- 第四行：第三幕（p.85-110）

每张卡片标注：+/-（这场戏对主角的情绪方向）+ 冲突（谁和谁，争什么，谁赢）

检验规则：某一行全是 + 或全是 - → 节奏单调需调整；每场戏必须有且只有一个冲突；找不到冲突 → 这场戏可能不需要存在。

### 第七步：提炼核心场景清单（McKee）

从节拍表中提取 5-8 个必须拍摄的关键场景。

**McKee 场景三要素**：
1. **价值反转（Turn）**：场景开始和结束时，主角的价值处境必须发生改变。没有反转 = 废戏。
2. **鸿沟（Gap）**：主角采取行动，期望得到某种回应，但现实给出相反结果。鸿沟越大，张力越强。
3. **转变方式**：场景只能通过两种方式转——行动（角色做了什么）或揭示（角色知道了什么）。

场景编号：
对应节拍：
一句话描述：

价值反转：开始 → 结束（正→负 / 负→正 / 负→更负）
转变方式：行动 / 揭示
鸿沟：主角期望___，现实给出___

情绪目标：观众应该感受到什么
进入钩子：前10秒用什么抓住观众
离开钩子：结束时留下什么未解决的张力
前提关联：这场戏如何推进/证明故事前提

### 第八步：提炼控制性理念（McKee）

从结局反推：控制性理念 = 结局带来的价值 + 导致这个结局的主要力量

控制性理念是前提的最终形态——故事开始时你有一个命题，结束时你证明了它。

### 输出格式（模式 A）

完成后输出五份文档：
1. Logline + 前提 + 类型定位
2. 人物三维骨架（所有核心角色，含 Save the Cat 时刻设计）
3. 节拍表（15节拍，含位置%和价值状态+/-）
4. 场景卡片概览（四行分布，标注+/-和冲突）
5. 核心场景清单（5-8场景，含价值反转和鸿沟）

## 模式 B：剧本优化

输入已有剧本或大纲，先诊断，再给方案。不直接改写，除非用户明确要求。

### 第一步：Logline 诊断

检验现有 logline 或故事概念是否通过 Snyder 四要素：讽刺性 / 视觉画面感 / 受众与成本 / 标题。

### 第二步：类型定位诊断

确认故事属于哪种 Snyder 类型，检验是否遵循该类型的核心规则（见 references/snyder-genres.md）。类型混乱是结构问题的常见根源。

### 第三步：结构诊断（Snyder 节拍表）

对照节拍表，逐一标注每个节拍的实际位置：

节拍 | 应在位置 | 实际位置 | 状态（正常 / 偏移 / 缺失）| 问题描述

常见结构病症：
- 催化剂太晚（>p.15）→ 开场拖沓
- 主角被动漂入第二幕 → 主角缺乏主体性
- 中段塌陷（娱乐与游戏段落无+/-变化）→ 故事停滞
- 中点太弱 → 第二幕后半无力
- 一败涂地不够惨 → 结局胜利没有重量
- 终场画面未呼应开场 → 主题没有视觉落点
- B故事没有在第三幕提供答案 → 顿悟缺乏情感根基

### 第四步：写作规则诊断（Snyder）

- **Double Mumbo Jumbo**：故事里有没有两个超自然前提？（观众只能接受一个"信念飞跃"）
- **Laying Pipe**：进入正题前是否铺垫过多？（催化剂前超过10页就危险）
- **Covenant of the Arc**：每个主要角色（除反派）是否都有改变？
- **Watch Out for that Glacier**：坏人逼近是否足够紧迫？（危险必须是当下的）
- **Pope in the Pool**：必要的信息传递是否用了有趣的场景来掩盖？

### 第五步：人物诊断（Egri）

三维骨架是否完整 / Want 和 Need 是否冲突 / 对立统一是否成立 / Save the Cat 时刻是否有效。

### 第六步：场景诊断（McKee）

抽取 3-5 个关键场景，检验价值反转和鸿沟是否存在。场景开始和结束价值状态相同 = 废戏。

### 第七步：输出改写方案

问题：具体描述
影响：对整体结构/人物/主题的影响
改写方向：具体可操作的建议
优先级：高 / 中 / 低

按优先级排序：结构性问题 > 人物问题 > 场景级问题。

## 常见陷阱

- **前提太宽泛**：「爱很重要」不是前提，「占有性的爱摧毁它试图保护的东西」才是
- **Logline 缺乏讽刺性**：没有内在反差的 logline 无法制造"然后呢"的冲动
- **类型混乱**：两种 Snyder 类型的规则互相冲突，导致结构无法自洽
- **主角被动**：主角必须主动选择进入第二幕，被动漂入的主角缺乏主体性
- **中段塌陷**：娱乐与游戏段落（20-50%）如果没有持续的+/-变化，故事停滞
- **一败涂地太轻**：主角必须失去他最在乎的东西，否则第三幕的胜利没有重量
- **B故事孤立**：B故事必须在第三幕转折时提供答案，否则它只是支线，不是主题载体
- **终幕画面忘记呼应开场**：开场和终幕是主题改变的视觉证明，缺少呼应 = 主题没有落地
