# Film Screenplay

剧本是叙事大纲和分镜之间的桥梁，也是整个创作过程中最需要反复迭代的文档。

方法论基础：
- **Trottier**（《编剧圣经》）：工业标准格式，五大元素，一页一分钟律条
- **McKee**（《对话》）：潜台词法则，台词即行动，三层内容球体
- **Akers**（《你的剧本烂透了》）：100条纠错清单，动作描写禁忌，自查机制

三种工作模式：
- **模式 A：从大纲生成草稿** — 输入节拍表/场景清单，生成可拍摄的剧本
- **模式 B：优化已有剧本** — 输入剧本雏形，诊断问题，迭代改写
- **模式 C：局部场景修改** — 分镜过程中发现问题，针对单个场景快速调整

---

## 格式规范（Trottier，输出时强制执行）

剧本格式是工业标准，不是风格选择。格式错误会让专业读者立刻失去信任。Trottier 说：场景标题是所有格式元素中错误最多的地方。

**五大元素**：

**① 场景标题（Scene Heading / Slugline）**

三部分结构：`[摄影机位置]. [地点] - [时间]`

```
INT. 四季茶馆 - 大堂 - 夜        ← 主场景标题（完整三部分）
大堂                              ← 次级场景标题（只写地点，无 INT./时间）
INT./EXT. 汽车 - 日              ← 跨越内外
```

**地点层级规则**（从大到小）：
- ✓ `INT. 四季酒店 - 大堂 - 夜`
- ✗ `INT. 大堂 - 四季酒店 - 夜`

**场景标题中禁止写描述**：
- ✗ `EXT. 月黑风高的小巷 - 夜`
- ✓ `EXT. 小巷 - 夜`，描述写进动作行

**时间标注**：DAY / NIGHT / CONTINUOUS（时间连续）/ SAME（同时发生）/ LATER（时间跳过）

**② 动作描写（Action Lines）**
- 现在时态，主动语态
- 每段不超过 4 行
- 只写摄影机能看到和听到的内容
- 人物首次出场时名字全大写，之后正常书写

**③ 角色名（Character Name）**
- 居中，全大写
- `(V.O.)`：画外音旁白 / `(O.S.)`：画外声音 / `(CONT'D)`：连续说话被动作打断后继续

**④ 对白（Dialogue）**
- 居中缩进，只写说出口的话

**⑤ 括号指示（Parenthetical）**
- 极度克制使用
- ✓ 指明说话对象 / 语言切换 / 朗读内容
- ✗ 描述情绪 / 描述表演方式

**特殊场景格式**（MONTAGE / SERIES OF SHOTS / INTERCUT / FLASHBACK / INSERT）详见 [references/screenplay-format.md](references/screenplay-format.md)。

## 模式 A：从大纲生成草稿

**输入**：阶段一的核心场景清单（含价值反转、鸿沟、情绪目标）

**生成步骤**（按顺序，不可跳过）：

**第一步：确认价值反转**
这场戏从什么状态开始，到什么状态结束？所有台词和动作必须驱动这个转变。

**第二步：设计场景入口**
第一个动作/台词用什么抓住观众？（对应场景清单的"进入钩子"）

**第三步：写动作描述**（Akers 标准）
- 现在时，主动语态，动态动词
- ✓ `他把杯子推到桌边，手没有离开`
- ✗ `他是悲伤的` / `我们看到他很难过`
- 每段 ≤ 4 行，一段一个动作节拍

**第四步：写台词**（McKee 标准）
每句台词必须是一个行动——推进价值反转、揭示 Want/Need、或制造鸿沟。
写完后执行**潜台词自查**（见下方）。

**第五步：设计场景出口**
最后一个动作/台词留下什么未解决的张力？

**输出格式**：

```
场景 [编号]：[场景标题]
价值反转：[开始状态] → [结束状态]
版本：v1

INT./EXT. [地点] - [时间]

[动作描写，≤4行/段，现在时主动语态]

角色名A
（括号指示，仅必要时）
台词内容

角色名B
台词内容

[关键动作节点]
```

## 模式 B：优化已有剧本

**输入**：用户提供的剧本雏形

**诊断维度**（按优先级排序）：

**① 格式层**（Trottier）：
- 场景标题格式是否正确？
- 动作描写是否超过 4 行？
- 括号指示是否滥用？
- 是否出现"我们看到/听到"？
- 是否使用了状态动词（is standing, are walking）？

**② 结构层**（McKee《故事》）：
- 价值反转是否发生？（没有 = 废戏）
- 鸿沟是否存在？
- 入口和出口钩子是否有力？

**③ 台词层**（McKee《对话》）：
- 有没有 on-the-nose 台词？（人物直接说出内心想法）
- 每句台词是否是一个行动？
- 台词是否有潜台词？（说的和想的是否有张力）
- 每个人物的声音是否独特？（换个名字还能认出是谁说的吗？）

**④ 动作层**（Akers）：
- 动作描写是否具体可拍？
- 是否用了动态动词？
- 道具是否被充分利用？

**输出格式**：

```
诊断报告：
格式问题：[列出具体问题]
结构问题：[价值反转/鸿沟/钩子]
台词问题：[列出具体问题台词 + 改写建议]
动作问题：[列出具体问题 + 改写建议]

修改版本（v[N+1]）：
[完整修改后的场景文本，严格遵循格式规范]

改动说明：[列出主要改动及原因]
```

## 模式 C：局部场景修改（分镜触发）

**触发场景**：分镜设计时发现台词/动作需要调整。

**处理原则**：
- 不需要走完整诊断流程
- 明确说明修改触发原因
- 修改后标注版本号
- 确认修改是否影响已设计的镜头

**输出格式**：

```
修改触发：[分镜发现的具体问题]
修改范围：场景 [编号]，[具体位置]

原版本（v[N]）：[原台词/动作]
修改版本（v[N+1]）：[新台词/动作]

修改原因：[为什么这样改]
对分镜的影响：[是否影响已设计的镜头]
```

## 台词潜台词自查（McKee《对白》，每次写完台词后执行）

完整规则见 [references/dialogue-craft.md](references/dialogue-craft.md)。

**核心前提：先判断对白类型**

写台词之前，必须先判断这是哪种对白：
- **叙述化对白**（旁白、独白、对观众说话）：允许直接告知，重点是速度和反衬，"show don't tell"不适用
- **戏剧化对白**（角色互相说话）：潜台词规则全面生效，以下自查适用

**三层球体（The Said / Unsaid / Unsayable）**：
- 说出的（The Said）：台词的字面内容
- 未说出的（The Unsaid）：人物想说但没说——潜台词的来源
- 无法说出的（The Unsayable）：人物自己都不知道的内心深处

三层都是同一个答案 → On-the-Nose，必须修复。

**自查八步**（逐句执行，详见 dialogue-craft.md）：

1. **判断对白类型**：叙述化 or 戏剧化？
2. **标注行动类型**：威胁/引诱/攻击/回避/安慰/投降/揭示/隐藏/说服/拒绝——说不出来考虑删除
3. **检查三层球体**：Said / Unsaid / Unsayable 是否有张力？
4. **检查权力转移**：对话开始和结束，谁更强？没有变化 → 废话
5. **检查冲突类型**：对称/不对称/间接/最小——是否符合场景情绪目标？
6. **检查声音独特性**：遮住角色名还能认出是谁说的吗？
7. **检查感官语言**：有没有可以用感官意象替换的抽象表达？
8. **检查病症**：不可信/陈词滥调/On-the-Nose/重复/过度解释/台词太长

## 动作描写自查（Akers，输出前执行）

**禁止清单**（出现即修改）：

| 禁止 | 原因 | 替换方式 |
|------|------|---------|
| `我们看到...` / `我们听到...` | 剧本是第三人称，不是导演指令 | 直接描述画面 |
| `摄影机推向...` / `特写：` | 导演决定镜头，编剧决定内容 | 通过动作描写暗示 |
| `他是悲伤的` / `她感到愤怒` | 情绪不可见，只有行动可见 | `他把照片翻过去` |
| `is standing` / `are walking` | 进行时削弱动感 | `stands` / `walks` |
| 超过 4 行的动作段落 | 读起来像小说，破坏节奏 | 拆分，每段一个动作节拍 |
| 括号里写情绪 `（愤怒地）` | 侵犯演员的工作 | 通过台词和动作表现 |

**动态动词原则**：
- ✗ `他走向门口` → ✓ `他冲向门口` / `他拖着脚步走向门口`
- ✗ `她看着他` → ✓ `她盯着他` / `她扫了他一眼，移开视线`

**Akers 故事层面检查**（模式 B 诊断时额外执行）：
- 主角有清晰的目标吗？（具体、可量化、有代价）
- 主角是主动的吗？（他在推动故事，还是被故事推着走？）
- 前十页抓住读者了吗？（建立主角、世界、核心冲突）
- 每个场景都在推进故事或揭示人物吗？（删掉后故事还成立 = 废场景）
- 反派够强吗？（反派越强，主角的胜利越有价值）

详细 Akers 纠错清单见 [references/dialogue-craft.md](references/dialogue-craft.md)。

## 剧本迭代原则

剧本不是一次写完的文档，是随创作推进持续演化的。

**正常迭代路径**：
- 大纲确定后写 v1（重在结构，台词粗糙）
- 分镜设计时发现问题，改成 v2（精细化台词和动作）
- 拍摄时灵感来了，现场调整成 v3

**需要大改的信号**（回到模式 B）：
- 价值反转消失了
- 人物说话不像这个人物了
- 场景和整体前提脱节了

**只需局部调整的信号**（模式 C）：
- 台词节奏问题
- 动作描写不够具体
- 分镜执行时发现的技术性问题
