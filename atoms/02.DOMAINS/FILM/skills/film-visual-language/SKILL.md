# Film Visual Language

两个使用模式，服务于不同阶段。

方法论基础：
- **Bruce Block**（《视觉故事》The Visual Story）：七个视觉组件 + 对比与亲和原则
- **Bellantoni**（《这就是紫色，有人要死》）：色彩心理学（已在 film-art-direction 中详述）

---

## 核心原则：对比与亲和（Contrast and Affinity）

Block 的核心操作原则，适用于所有七个视觉组件：

- **高对比（High Contrast）**：两个视觉元素差异大 → 制造张力、冲突、不安、戏剧性
- **高亲和（High Affinity）**：两个视觉元素相似 → 制造和谐、平静、统一、压抑（过度亲和）

**关键洞察**：对比和亲和不只是美学选择，而是叙事工具。场景的情绪目标决定应该用高对比还是高亲和。

---

## 七个视觉组件（Bruce Block）

每个组件都可以独立控制，也可以组合使用：

| 组件 | 定义 | 对比效果 | 亲和效果 |
|------|------|---------|---------|
| **空间（Space）** | 画面的深度感：深空间 vs 浅空间 | 深浅对比 → 张力、层次 | 统一深度 → 平静或压抑 |
| **线条（Line）** | 画面中的线条方向：水平/垂直/斜线/曲线 | 线条冲突 → 不稳定 | 线条统一 → 秩序或单调 |
| **形状（Shape）** | 画面中的几何形态：有机/几何/不规则 | 形状对比 → 冲突 | 形状重复 → 节奏感 |
| **色调（Tone）** | 画面的明暗分布：高调/低调/中间调 | 明暗对比 → 戏剧性 | 均匀色调 → 平淡或压抑 |
| **色彩（Color）** | 色相、饱和度、色温 | 互补色对比 → 张力 | 类似色 → 和谐 |
| **运动（Movement）** | 人物/摄影机/剪辑的运动方向和速度 | 运动方向冲突 → 混乱 | 运动方向统一 → 流畅 |
| **节奏（Rhythm）** | 视觉元素的重复和变化频率 | 节奏打破 → 惊喜/冲击 | 节奏稳定 → 预期/催眠 |

## 模式一：全片定调（项目开始时用一次）

输入：节拍表 + 核心场景清单 + 人物三维骨架。

### 第一步：确定全片视觉基调

从叙事前提出发，确定这部影片的整体视觉策略：

**核心问题**：这部影片的视觉语言应该主要使用高对比还是高亲和？

- 高对比为主：冲突激烈、道德模糊、心理紧张的故事
- 高亲和为主：压抑、克制、情绪内敛的故事
- 混合策略：日常段落高亲和，冲突节点高对比（最常见）

### 第二步：七组件基准设定

对每个视觉组件，定义全片的默认状态和转变触发条件：

**空间基准**：
- 默认深度：深空间（有纵深层次）/ 浅空间（压缩、平面感）
- 转变触发：什么叙事事件驱动空间深度变化？

**线条基准**：
- 主导线条方向：水平（平静/死亡）/ 垂直（权威/压迫）/ 斜线（不稳定/动荡）/ 曲线（柔和/流动）
- 线条冲突触发：什么时候引入对立线条方向？

**色调基准**：
- 默认调性：高调（明亮，喜剧/梦幻）/ 低调（暗部为主，紧张/危险）/ 中间调（中性）
- 转变触发：什么节拍驱动色调转变？

**色彩基准**：
- 基准色温：暖（亲密/危险）/ 冷（疏离/死亡）/ 中性
- 基准饱和度：高饱和（情绪强烈）/ 低饱和（压抑/现实）/ 去饱和（极度压抑）
- 色彩转变节点：对照节拍表，标注哪些节拍需要色彩变化，说明叙事理由

**运动基准**：
- 摄影机默认状态：静止 / 运动（跟随 / 独立运动）
- 运动的叙事意图：摄影机开始运动意味着什么？
- 手持 vs 稳定：手持传递不稳定感，稳定传递冷静或压迫

**节奏基准**：
- 默认剪辑节奏：镜头平均时长参考
- 加速触发：什么叙事事件驱动节奏加快
- 减速触发：什么叙事事件驱动节奏放慢

### 第三步：视觉动机（Visual Motif）

视觉动机是贯穿全片的视觉符号，在不同场景中反复出现并积累意义。

```
动机 1：[视觉元素] → [初次出现时的含义] → [最终出现时的含义变化]
动机 2：[视觉元素] → [含义] → [含义变化]
```

视觉动机可以是：特定颜色、特定形状、特定构图模式、特定运动方向、特定道具。

### 第四步：景别变化逻辑

不是定"全片用什么景别"，而是定"景别变化的叙事触发条件"：

- 景别收紧触发（中景→近景→特写）：什么情绪状态或叙事事件？
- 景别放开触发（特写→中景→全景）：什么叙事事件？
- 特写策略：克制还是频繁？克制意味着特写出现时有更大的重量

**输出格式**：

```
全片视觉基调：[高对比为主 / 高亲和为主 / 混合策略]

七组件基准：
- 空间：[默认深度] | 转变触发：[描述]
- 线条：[主导方向] | 冲突触发：[描述]
- 色调：[默认调性] | 转变触发：[描述]
- 色彩：[色温+饱和度] | 转变节点：[节拍X：原因]
- 运动：[默认状态+意图]
- 节奏：[基准时长] | 加速触发：[描述] | 减速触发：[描述]

视觉动机：
- [动机1]：[含义轨迹]
- [动机2]：[含义轨迹]

景别逻辑：
- 收紧触发：[描述]
- 放开触发：[描述]
- 特写策略：[克制/频繁，说明]
```

## 模式二：分镜校验（每次做分镜时调用）

输入：当前场景的叙事目的 + 已建立的视觉决策框架。

对每个镜头的视觉决策逐项检查，重点是**叙事理由**，而不是机械打勾：

**对比/亲和检查**
- 这个镜头的视觉对比度（高/低）是否符合场景的情绪目标？
- 如果场景需要张力，是否有足够的对比？如果需要压抑，是否有足够的亲和？

**七组件检查**（只检查与当前场景最相关的组件）
- 空间深度：是否符合基准？偏离的叙事理由？
- 线条方向：是否引入了对立线条？为什么？
- 色调：明暗分布是否符合当前节拍的色调基准？
- 色彩：色温和饱和度是否符合当前节拍的色彩状态？
- 运动：摄影机运动是否有叙事动机？
- 节奏：当前场景的剪辑节奏是否符合基准？触发条件是否成立？

**视觉动机检查**
- 这个场景是否应该出现视觉动机？
- 如果出现，动机的状态是否反映了当前的叙事状态？

**一致性检查**
- 这个镜头的视觉决策和前面已完成的分镜是否在同一部影片的语言体系里？
- 有没有无意识的风格漂移？

**输出格式**：

```
场景：[场景名] | 节拍位置：[节拍编号]

对比/亲和：[高/低] | 符合情绪目标：[是/否，原因]

组件检查：
- 色调：[符合/偏离] | 偏离原因：[如有]
- 色彩：[符合/偏离] | 偏离原因：[如有]
- 运动：[符合/偏离] | 叙事动机：[描述]
- 节奏：[符合/偏离] | 触发条件：[描述]

视觉动机：[出现/未出现] | 状态：[描述]
一致性：[符合/偏离] | 风格漂移风险：[描述]
```

## 模式三：框架修订（发现全片框架需要调整时触发）

**触发场景**：
- 分镜执行中发现某个基准设定让场景失去叙事效果
- 用户看了分镜稿后觉得整体视觉方向需要调整
- 拍摄时灵感来了，想修改某个组件的全片策略

**处理原则**：
- 框架修订不是失败，是创作迭代的正常部分
- 修订前先判断：是这个场景的特殊需求（局部偏离），还是全片框架本身有问题（需要修订）
- 局部偏离：在模式二的输出中记录叙事理由，不修改框架
- 全片修订：走模式三，更新基准，标注版本号，说明修订原因

**修订流程**：

**第一步：诊断问题**
- 哪个组件的基准设定有问题？
- 问题是"基准本身错了"还是"基准对大多数场景有效，但这个场景是例外"？
- 如果是例外，用局部偏离处理，不修订框架

**第二步：评估影响范围**
- 修改这个基准，会影响哪些已完成的分镜？
- 已完成的分镜需要重新校验吗？

**第三步：更新框架**
- 修改对应的组件基准
- 标注版本号（v1 → v2）
- 记录修订原因和修订时间（对应哪个场景触发了修订）

**输出格式**：

```
修订触发：[哪个场景/什么原因触发了修订]
修订范围：[哪个组件的哪个基准]

原基准（v[N]）：[原来的设定]
新基准（v[N+1]）：[修订后的设定]

修订原因：[为什么原来的设定有问题]
影响范围：[哪些已完成的分镜需要重新校验]
```
