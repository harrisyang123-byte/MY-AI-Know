# SubAgent 配置 (SubAgents Configuration)

## 3个专业子代理

### SubAgent 1: image_sub_agent

**定位**：图像生成/编辑专家

**核心能力**：
- 图像生成（单图/多图）
- 海报设计
- 演示文稿
- 批量图像生成

**委托场景**：
- 多图批量生成
- 海报设计（需要多元素组合）
- 演示文稿（多页设计）
- 需要多次迭代的复杂图像任务

**委托时必须提供**：
- 完整项目上下文
- 所有参考图像URL
- 用户的完整需求
- 设计重点
- 文本/数据内容

---

### SubAgent 2: video_sub_agent

**定位**：视频生成/编辑专家

**核心能力**：
- 视频生成
- 视频编辑
- BGM生成
- 视频处理

**委托场景**：
- 视频生成（从文本/图像）
- 视频编辑（裁剪/拼接/特效）
- BGM生成
- 视频格式转换

**委托时必须提供**：
- 视频需求描述
- 参考素材（图像/视频URL）
- 风格和氛围要求
- 时长和分辨率要求
- BGM风格（如需要）

---

### SubAgent 3: three_d_sub_agent

## 委托规则

### 何时委托

| 场景 | 委托给 |
|------|--------|
| 多图批量生成 | image_sub_agent |
| 海报设计 | image_sub_agent |
| 演示文稿 | image_sub_agent |
| 视频生成/编辑 | video_sub_agent |
| BGM生成 | video_sub_agent |
| 3D模型生成 | three_d_sub_agent |

### 何时自己做

| 场景 | 原因 |
|------|------|
| 单图生成 | 直接调用工具即可 |
| 单图编辑 | 直接调用编辑工具 |
| 图像调整（比例/分辨率） | 直接调用reframe/upscale |
| 简单迭代优化 | 保持上下文连贯性 |

### 委托原则

1. **完整性**：委托时必须提供完整的项目上下文，包括所有参考图像URL、用户需求、设计重点
2. **一次性**：多个输出一次调用，不连续调用同一代理
3. **上下文传递**：将当前对话中的关键信息（场景描述、风格偏好、约束条件）完整传递给SubAgent
4. **结果验证**：SubAgent返回结果后，验证是否符合需求，必要时进行迭代优化

### 委托模板

```
委托给: [SubAgent名称]
项目上下文:
  - 项目目标: [...]
  - 核心约束: [...]
  - 整体风格: [...]
  - 用户偏好: [...]
参考图像:
  - [URL1]: [描述]
  - [URL2]: [描述]
具体任务:
  - [任务1]
  - [任务2]
设计重点:
  - [重点1]
  - [重点2]
```
