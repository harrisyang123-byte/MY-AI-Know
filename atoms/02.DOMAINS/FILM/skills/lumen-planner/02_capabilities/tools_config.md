# 工具配置 (Tools Configuration)

## 图像生成工具

### Tool 1: generate_image_gpt_image_2

**定位**：默认首选，通用全能型

**核心优势**：
- 指令跟随能力强
- 通用场景表现好
- 质量可控 (auto/high/medium/low)
- 尺寸灵活 (精确到16的倍数)
- 创意与控制平衡好

**适用场景**：
- 概念视觉设计
- 产品级图形输出
- 需要精确控制的任务
- 复杂构图和布局
- 多元素组合场景

**参数配置**：

| 参数 | 说明 | 规则 |
|------|------|------|
| size | WxH格式 | W/H必须是16的倍数，最长边<3840，比例<=3:1，像素范围[655360, 8294400] |
| quality | auto/high/medium/low | high: 用户明确要求最高质量；medium: 默认平衡；low: 快速草稿迭代预览；auto: 模型决定 |
| image_url_list | 参考图像列表 | 第一张通常是主图，其他是参考图 |

**常用尺寸**：

| 比例 | 尺寸 |
|------|------|
| 9:16 | 1440x2560 |
| 16:9 | 2560x1440 |
| 1:1 | 2048x2048 |
| 3:2 | 2400x1600 |

**使用原则**：
- 没有强烈理由选其他工具时，默认用这个
- 需要精确控制时，首选这个
- 质量默认medium，除非用户明确要求

**不适用场景**：
- 极致艺术风格 → 用Midjourney
- 动漫/东方美学 → 用Midjourney niji7

### Tool 2: generate_image_nano_banana_2

**定位**：多分辨率支持，任务类型明确

**核心优势**：
- 多分辨率支持 (512/1K/2K/4K)
- 任务类型分类清晰
- 适合简单到中等复杂度任务

**适用场景**：
- 需要特定分辨率输出
- 任务类型明确的场景
- 文本到图像生成
- 参考图像生成
- 单图编辑
- 多图融合

**参数配置**：

| 参数 | 说明 | 规则 |
|------|------|------|
| resolution | 512/1K/2K/4K | 默认1K，仅用户明确要求时使用更高分辨率 |
| aspect_ratio | 预设比例 | 8:1, 4:1, 21:9, 16:9, 3:2, 4:3, 5:4, 1:1, 4:5, 3:4, 2:3, 9:16, 1:4, 1:8；编辑任务设为最接近原图比例 |
| task_type | 任务类型 | TEXT_TO_IMAGE / REFERENCE_TO_IMAGE / EDIT_SINGLE_IMAGE / FUSION_MULTI_IMAGES / OTHER |

**使用原则**：
- 用户明确需要特定分辨率时
- 任务类型清晰明确时
- 需要快速生成时

**不适用场景**：
- 需要极致质量控制
- 复杂的多元素场景

### Tool 3: generate_image_nano_banana_pro

**定位**：单图任务，文本渲染强

**核心优势**：
- 文本渲染能力强
- 适合简单单图任务
- 比例灵活
- 用户意图保留好

**适用场景**：
- 需要渲染文字的图像 (Logo/图表/海报)
- 简单的单图生成
- 用户需求非常明确清晰
- 文本翻译任务

**参数配置**：

| 参数 | 说明 | 规则 |
|------|------|------|
| aspect_ratio | 预设比例 | 21:9, 16:9, 4:3, 3:2, 1:1, 9:16, 3:4, 2:3, 5:4, 4:5；编辑/参考任务设为最接近原图比例 |
| resolution | 1K/2K/4K | 默认1K，仅用户明确要求时使用更高分辨率 |
| task_type | 任务类型 | 同nano_banana_2 |
| prompt | 提示词 | 用户需求清晰时直接使用用户原话，不添加修改或解释用户意图 |

**使用原则**：
- 仅用于简单单图任务
- 用户未指定其他工具时
- 需要文本渲染时优先考虑

**不适用场景**：
- 复杂场景
- 多元素组合
- 需要精确控制的任务

---

### Tool 4: generate_image_midjourney

**定位**：艺术风格，视觉冲击

**核心优势**：
- 艺术表现力极强
- 视觉效果震撼
- 戏剧性光影和质感
- 风格化和幻想场景
- 动漫/东方美学 (niji7模型)

**适用场景**：
- 艺术创作
- 概念设计
- 风格化插画
- 幻想/科幻场景
- 动漫角色和场景
- 东方美学风格

**参数配置**：

| 参数 | 说明 | 规则 |
|------|------|------|
| aspect_ratio | 宽高比 | 格式"width:height"，支持:*和x作为分隔符 |
| is_anime_or_oriental_aesthetic_style | 布尔值 | true: 使用niji7模型；false: 使用v7模型(默认) |
| image_url | 图生图 | 提供源图像URL进行转换 |
| sref | 风格参考种子 | 用于风格一致性 |
| prompt | 提示词 | 短而简单最好，避免长列表和详细指令，使用具体描述性词汇 |

**niji7特性**：
- 增强角色控制
- 风格灵活性（写实到高度风格化）
- 深度文本理解
- 多角色一致性

**Midjourney提示词规则**：
- 短而简单最好
- 避免长列表和详细指令
- 使用具体描述性词汇
- 具体数字而非复数（"三只猫" vs "猫"）
- 关注想要的，而非不想要的
- 考虑：主体/媒介/环境/光照/色彩/情绪/构图

**使用原则**：
- 艺术风格优先于技术精确时
- 需要视觉冲击力时
- 动漫/东方美学任务

**不适用场景**：
- 需要精确文本渲染
- 需要完美解剖结构（手/脸）
- 严格布局和透视要求
- 产品图等需要精确控制的场景

## 图像编辑工具

### Tool 5: reframe_image

**定位**：改变比例/尺寸

**核心能力**：
- 改变图像比例
- 调整图像尺寸
- 智能路由：
  - 相同比例缩放：PIL缩小 / AI放大
  - 比例改变（匹配支持比例）：Nano Banana Pro重生成
  - 比例改变（其他）：Seedream V4 AI扩展

**参数配置**：

| 参数 | 说明 | 规则 |
|------|------|------|
| expected_ratio | 期望比例 | 格式"width:height"（如"16:9"），与expected_size二选一 |
| expected_size | 期望尺寸 | 格式"width*height"（如"1920*1080"），与expected_ratio二选一 |
| image_url | 源图像URL | 必填 |

**使用原则**：
- 只用expected_ratio或expected_size，不要同时用
- 不要传递字符串"None"

## 工具选择决策树

```
任务类型?
  │
  ├─ 图像生成
  │   │
  │   ├─ 用户指定工具? → 使用指定工具
  │   │
  │   ├─ 艺术/动漫/东方美学风格? → generate_image_midjourney
  │   │
  │   ├─ 需要文本渲染? → generate_image_nano_banana_pro
  │   │
  │   ├─ 简单单图任务? → generate_image_nano_banana_pro
  │   │
  │   └─ 其他 → generate_image_gpt_image_2 (默认)
  │
  ├─ 图像编辑
  │   │
  │   ├─ 改变比例/尺寸? → reframe_image
  │   │
  │   └─ 提升分辨率? → upscale_image
  │
  ├─ 视频生成 → call_subagent(video_sub_agent)
  │
  ├─ 3D模型 → call_subagent(three_d_sub_agent)
  │
  └─ HTML页面 → generate_html (仅当用户说"HTML")
```

---

## 关键规则

1. 有图像 → 先 analyse_image
2. 没有强烈理由 → 用 generate_image_gpt_image_2
3. 用户指定工具 → 优先使用
4. 提示词 → 用英语
5. 响应 → 用用户语言
6. 多个输出 → 一次调用
7. 不要同时调用 retrieve_skills 和 search_image
