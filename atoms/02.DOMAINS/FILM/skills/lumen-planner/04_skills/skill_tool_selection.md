# Skill 5: Tool Selection (工具选择)

## 核心理念

### 什么是 Tool Selection？

在正确的时机，选择正确的工具，用正确的方式。

**不是**：
- "我有锤子，所以一切都是钉子"
- "我就用这个工具，不管合不合适"
- "随便选一个试试"

**而是**：
- 分析任务需求
- 匹配工具能力
- 选择最优方案
- 优化参数配置

### 为什么重要？

选对工具 = 事半功倍，选错工具 = 事倍功半

**示例**：
- 任务：生成一张产品图
- ❌ 用Midjourney → 很美，但产品细节不对
- ✅ 用GPT-Image-2 → 准确还原产品需求

| **指令跟随** | ★★★★★ | ★★★ | ★★★ | ★★ |
| **艺术表现** | ★★★ | ★★★ | ★★★ | ★★★★★ |
| **文本渲染** | ★★★ | ★★★ | ★★★★★ | ★ |
| **控制精度** | ★★★★★ | ★★★ | ★★★ | ★★ |
| **尺寸灵活** | ★★★★★ | ★★★★ | ★★★ | ★★★ |
| **速度** | ★★★ | ★★★★ | ★★★★ | ★★★ |

### 决策流程

```
Step 1: 用户指定工具?
  │
  ├─ 是 → 使用指定工具 (用户最优先)
  │
  └─ 否 → Step 2
           │
           ├─ 艺术风格优先?
           │   ├─ 动漫/东方美学? → midjourney (niji7)
           │   └─ 其他艺术风格? → midjourney (v7)
           │
           ├─ 需要文本渲染? → nano_banana_pro
           │
           ├─ 简单单图任务? → nano_banana_pro
           │
           ├─ 需要特定分辨率? → nano_banana_2
           │
           └─ 其他 → gpt_image_2 (默认)
```

## 各工具详细选择指南

### 何时选择 gpt_image_2

**选择条件**（满足任一）：
- 没有强烈理由选其他工具
- 需要精确控制（复杂构图、多元素组合）
- 概念视觉设计
- 产品级图形输出
- 需要参考图像生成

**参数优化**：

| 场景 | quality | size | 说明 |
|------|---------|------|------|
| 默认 | medium | 2048x2048 | 平衡质量和速度 |
| 高质量需求 | high | 2048x2048 | 用户明确要求 |
| 快速草稿 | low | 1024x1024 | 迭代预览 |
| 海报 | medium | 1440x2560 | 竖版9:16 |
| 风景 | medium | 2560x1440 | 横版16:9 |

---

### 何时选择 nano_banana_2

**选择条件**（满足任一）：
- 用户明确需要特定分辨率
- 任务类型清晰明确
- 简单到中等复杂度
- 需要快速生成

**参数优化**：

| 场景 | resolution | aspect_ratio | task_type |
|------|-----------|--------------|-----------|
| 默认 | 1K | 1:1 | TEXT_TO_IMAGE |
| 高清需求 | 2K/4K | 按需 | TEXT_TO_IMAGE |
| 参考图生成 | 1K | 接近原图 | REFERENCE_TO_IMAGE |
| 单图编辑 | 1K | 接近原图 | EDIT_SINGLE_IMAGE |
| 多图融合 | 1K | 按需 | FUSION_MULTI_IMAGES |

---

### 何时选择 nano_banana_pro

**选择条件**（满足任一）：
- 需要渲染文字的图像（Logo/图表/海报）
- 简单的单图生成
- 用户需求非常明确清晰
- 文本翻译任务

**参数优化**：

| 场景 | resolution | aspect_ratio | prompt策略 |
|------|-----------|--------------|-----------|
| 默认 | 1K | 1:1 | 用户原话 |
| 海报 | 1K | 9:16 | 用户原话 |
| Logo | 1K | 1:1 | 用户原话 |

**关键规则**：
- 用户需求清晰时：直接使用用户原话
- 不要添加、修改或解释用户意图
- 完全保留用户原始意图

---

### 何时选择 midjourney

**选择条件**（满足任一）：
- 艺术风格优先于技术精确
- 需要视觉冲击力
- 动漫/东方美学任务
- 风格化插画
- 幻想/科幻场景

**参数优化**：

| 场景 | is_anime | sref | prompt策略 |
|------|----------|------|-----------|
| 动漫/东方美学 | true | 按需 | 短而简单 |
| 艺术风格 | false | 按需 | 短而简单 |
| 风格一致性 | 按需 | 使用 | 短而简单 |

**Midjourney提示词规则**：
- 短而简单最好
- 避免长列表和详细指令
- 使用具体描述性词汇
- 具体数字而非复数
- 关注想要的，而非不想要的
- 考虑：主体/媒介/环境/光照/色彩/情绪/构图

## 图像编辑工具选择

### reframe_image vs upscale_image

| 需求 | 工具 | 说明 |
|------|------|------|
| 改变比例 | reframe_image | 从16:9改成9:16等 |
| 改变尺寸 | reframe_image | 从2048x2048改成1024x1024等 |
| 提升分辨率 | upscale_image | 从1K提升到4K |
| 社交媒体适配 | reframe_image | 不同平台比例 |
| 打印需求 | reframe_image | 特定尺寸 |

## 常见错误与避免

### 错误1：默认工具不是最优

**场景**：用户要动漫风格，你用gpt_image_2
**问题**：gpt_image_2的动漫表现不如midjourney niji7
**解决**：艺术风格优先时，考虑midjourney

### 错误2：过度使用Midjourney

**场景**：产品图用midjourney
**问题**：艺术性太强，控制力不够，产品细节丢失
**解决**：需要精确控制时，用gpt_image_2

### 错误3：忽略用户指定

**场景**：用户说"用midjourney"，你用gpt_image_2
**问题**：用户指定优先级最高
**解决**：用户指定工具 → 直接使用

### 错误4：参数配置不当

**场景**：简单任务用high quality
**问题**：浪费时间，速度慢
**解决**：简单任务用medium/low，高质量需求才用high

## 贯穿案例：清代海军衙门办公室工具选择

### 场景分析

```
任务: 生成清代海军衙门办公室图像
需求特征:
  - 复杂空间布局(8m x 6m房间)
  - 多元素组合(门/窗/桌/椅/人物)
  - 精确光线控制(烛光95%+月光5%)
  - 写实电影风格
  - 需要参考图保持一致性
```

### 工具选择推理

```
Step 1: 用户指定工具?
  → 否

Step 2: 艺术风格优先?
  → 否。用户要"写实电影感"，不是艺术风格
  → 写实 ≠ 艺术风格

Step 3: 需要文本渲染?
  → 否。场景中没有文字渲染需求

Step 4: 简单单图?
  → 否。复杂构图+多元素+精确光线

Step 5: 需要特定分辨率?
  → 否。标准尺寸即可

Step 6: 默认 → gpt_image_2
  → 理由: 指令跟随★★★★★ + 控制精度★★★★★
  → 复杂7层提示词需要强指令跟随能力
  → 精确空间/光线描述需要高控制精度
```

### 参数配置

```
工具: generate_image_gpt_image_2
quality: medium (平衡质量和速度)
size: 2560x1440 (横版16:9，电影感)
prompt: [7层架构完整提示词]
```

### 后续角度的工具选择

```
第2张(西向东拍):
  → 同样用gpt_image_2
  → 使用第1张作为参考图(链式参考)
  → 保持一致性

细节特写(桌面/门框):
  → 同样用gpt_image_2
  → 更高quality(high)
  → 更小size聚焦细节
```

### 如果用户要求不同风格

```
用户: "能不能用水彩风格?"
  → 风格变为艺术风格优先
  → 考虑midjourney
  → 但空间描述复杂，midjourney控制力不足
  → 决策: 仍用gpt_image_2，但在第7层强调水彩风格
  → 理由: 空间精确性 > 风格艺术性

用户: "能不能用动漫风格?"
  → 动漫风格 → midjourney niji7
  → 提示词简化为短描述
  → 放弃7层架构的精确控制
  → 理由: 动漫风格midjourney专长，空间精确性在动漫中不重要
```

## 工具选择检查清单

每次选择工具前，检查以下项目：

- □ 用户是否指定了工具？（用户指定优先）
- □ 任务是否需要艺术风格优先？（考虑midjourney）
- □ 任务是否需要文本渲染？（考虑nano_banana_pro）
- □ 任务是否是简单单图？（考虑nano_banana_pro）
- □ 任务是否需要特定分辨率？（考虑nano_banana_2）
- □ 任务是否涉及复杂构图/多元素？（默认gpt_image_2）
- □ 任务是否需要精确控制？（默认gpt_image_2）
- □ 参数配置是否匹配场景？（quality/size/resolution）
- □ 是否需要委托SubAgent？（批量/视频/3D）
- □ 是否需要使用参考图？（链式参考策略）
