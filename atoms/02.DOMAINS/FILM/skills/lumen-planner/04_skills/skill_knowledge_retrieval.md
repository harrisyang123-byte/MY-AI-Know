# Skill 7: Knowledge Retrieval (知识检索)

## 核心理念

### 什么是 Knowledge Retrieval？

在正确的时机，用正确的方式，获取正确的信息。

**不是**：
- "遇到不懂的就搜"
- "搜一堆信息堆给用户"
- "盲目依赖搜索"

**而是**：
- 判断是否需要检索
- 选择合适的检索工具
- 提取关键信息
- 整合到任务中

### 为什么重要？

现实：
- 我的知识有限（截止日期）
- 用户需求千变万化
- 专业领域信息复杂
- 品牌/产品信息需要准确

解决方案：
- 通过检索补充知识
- 获取最新信息
- 获取专业信息
- 获取视觉参考

---

## 5种检索工具

| 工具 | 功能 | 适用场景 |
|------|------|---------|
| search_web | 搜索网络文本信息 | 事实、概念、背景信息 |
| search_image | 搜索参考图片 | 视觉参考、风格参考、灵感 |
| search_website_content | 提取网页内容 | 深度信息、官网内容、详细资料 |
| retrieve_skills | 检索设计技能指导 | 设计流程、最佳实践、专业指导 |
| explore_kits | 探索品牌/项目资源包 | 品牌资产、设计规范、素材库 |

---

## 检索决策框架

### 何时需要检索？

**场景1：知识盲区**

信号：
- 用户提到我不熟悉的概念
- 专业术语
- 特定品牌/产品
- 地域文化

示例：
- "设计一个赛博朋克风格的海报" → 我知道赛博朋克 → 不需要搜索
- "设计一个Wabi-sabi风格的海报" → 我不太确定Wabi-sabi → 需要搜索

行动：
→ 使用 search_web 了解概念
→ 使用 search_image 获取视觉参考

---

**场景2：时效性信息**

信号：
- 最新产品
- 当前趋势
- 实时数据
- 最新事件

示例：
- "设计iPhone 16 Pro的海报" → 需要最新产品信息 → 需要搜索
- "设计2024年流行的渐变风格" → 需要了解当前趋势 → 需要搜索

行动：
→ 使用 search_web 获取最新信息
→ 使用 search_image 获取最新视觉趋势

---

**场景3：专业领域**

信号：
- 特定行业
- 专业标准
- 技术规范
- 行业惯例

示例：
- "设计一个医疗产品包装" → 医疗包装有特殊规范 → 需要搜索
- "设计亚马逊产品主图" → 亚马逊有特定要求 → 需要检索技能

行动：
→ 使用 search_web 了解行业标准
→ 使用 retrieve_skills 获取专业指导

---

**场景4：视觉参考需求**

信号：
- 用户要求特定风格
- 需要灵感
- 需要参考案例
- 不确定视觉方向

示例：
- "设计一个日式极简风格的海报" → 需要视觉参考 → 需要搜索图片
- "给我一些产品摄影的灵感" → 需要参考案例 → 需要搜索图片

行动：
→ 使用 search_image 获取视觉参考

---

**场景5：品牌/项目资源**

信号：
- 用户提供 Kit ID
- 提到品牌规范
- 需要品牌资产
- 系列设计

示例：
- "用我的品牌Kit设计海报 --kit='xxx'" → 需要品牌资源 → 需要探索Kit

行动：
→ 使用 explore_kits 获取品牌资源

---

### 何时不需要检索？

**避免过度检索**：

| 场景 | 判断 | 行动 |
|------|------|------|
| 通用设计任务 | "简约风格"是通用概念 | 直接生成 |
| 我已经知道的信息 | 我知道Instagram规格和风格 | 直接生成 |
| 用户已提供足够信息 | 用户提供了参考图 | 分析图片，直接生成 |
| 创意探索阶段 | "给我设计几个不同风格的Logo" | 直接生成多个版本 |

**原则**：
- 不要为了搜索而搜索
- 不要过度依赖搜索
- 信任自己的知识和能力
- 只在真正需要时检索

## 工具使用指南

### search_web — 网络文本搜索

**功能**：搜索网络文本信息，获取事实、概念、背景知识

**适用场景**：
- 不熟悉的概念/术语
- 品牌/公司/产品信息
- 时效性信息
- 专业领域知识
- 文化/地域背景

**使用技巧**：

**技巧1：原子化查询**

- ✗ 错误："Wabi-sabi风格的特点和在现代设计中的应用"
- ✓ 正确："Wabi-sabi design style"
- 原则：一次只搜一个核心概念

**技巧2：精确关键词**

- ✗ 模糊："那个日本的设计风格"
- ✓ 精确："Japanese minimalism design"
- 原则：使用准确的术语

**技巧3：多次搜索**

如果需要多个概念，分别搜索，不要合并查询：

- ✗ 错误：一次搜"赛博朋克霓虹灯效果"
- ✓ 正确：
  - 第一次："cyberpunk design style"
  - 第二次："neon light effects"

---

### search_image — 图片搜索

**功能**：搜索参考图片，获取视觉参考和灵感

**适用场景**：
- 需要视觉参考
- 风格参考
- 灵感收集
- 案例研究
- 不确定视觉方向

**biz_type选择**：

| biz_type | 名称 | 使用场景 | 查询风格 |
|----------|------|---------|---------|
| 7 | WebSearch (网络搜索) | 实时/流行内容、特定品牌/产品、明确目标 | 具体品牌/产品名 |
| 8 | Boutique (精品设计库) | 设计参考、风格灵感、retrieve_skills之后 | 1-2个核心名词 |

**使用技巧**：

**技巧1：简洁查询**

- ✗ 错误："modern minimalist poster design with geometric shapes"
- ✓ 正确："minimalist poster"
- 原则：1-2个核心名词

**技巧2：核心概念**

- ✗ 错误："beautiful sunset landscape photography"
- ✓ 正确："sunset landscape"
- 原则：去掉形容词，保留核心名词

**技巧3：单一概念**

- ✗ 错误："cyberpunk neon city"
- ✓ 正确："cyberpunk city"
- 原则：一次搜一个概念

**关键规则**：
- 不要同时调用 retrieve_skills 和 search_image
- 先检索技能，技能中包含搜索策略
- 按技能指导进行后续搜索

---

### search_website_content — 网页内容提取

**功能**：提取网页内容（文本+图片），返回相关信息

**适用场景**：
- 用户提供了网站URL
- 需要深度信息
- 品牌官网信息
- 多模态分析
- 综合信息

---

### retrieve_skills — 设计技能检索

**功能**：检索设计技能指导，获取设计流程、最佳实践、专业指导

**适用场景**：
- 任务匹配可用技能
- 用户选择了技能（skill ID）
- 需要设计指导

**可用技能**：
- amazon_product_image（亚马逊产品图）
- instagram_post（社交媒体帖子）
- branding_logo（品牌Logo）
- brochure_design（宣传册设计）
- storyboard_design（故事板设计）
- carousel_design（轮播图设计）

**关键规则**：
- 不要同时调用 retrieve_skills 和 search_image
- 先检索技能，技能中包含搜索策略

---

### explore_kits — Kit资源探索

**功能**：探索品牌/项目资源包，获取品牌资产、设计规范、素材库

**适用场景**：
- 用户消息中有 --kit="kit_id"
- 需要品牌资产
- 系列设计

**使用流程**：
1. 检测用户消息中的 --kit="kit_id"
2. 调用 explore_kits 获取品牌资产
3. 提取资产（Logo/颜色/字体/指南）
4. 作为参考图像使用
5. 应用到生成中

**Kit资源类型**：
- 品牌Kit：Logo、品牌色、字体、设计指南
- 游戏Kit：角色、场景、道具
- 视频Kit：故事板、脚本、素材

---

## 检索结果整合流程

### Step 1：提取关键信息

从检索结果中提取与当前任务相关的关键信息

### Step 2：验证信息准确性

- 交叉验证（多个来源确认）
- 时效性检查（信息是否过时）
- 相关性检查（信息是否与任务相关）

### Step 3：整合到上下文

- 将新信息整合到3层上下文模型
- 检查与现有信息的冲突
- 更新项目层/任务层/交互层

### Step 4：转化为提示词

- 将检索到的知识转化为提示词元素
- 将视觉参考转化为参考图像
- 将专业指导转化为设计约束

### Step 5：应用到生成

- 在提示词中应用检索结果
- 使用参考图像指导生成
- 遵循专业指导的约束

## search_website_content 详细指南

### 功能

提取网页内容（文本+图片），返回与查询相关的信息

### 适用场景

- 用户提供了网站URL
- 需要深度品牌/产品信息
- 需要从官网提取设计规范
- 需要多模态内容分析（网页中的图片+文字）
- 需要综合信息整合

### 使用流程

```
1. 接收URL
   │
2. 调用 search_website_content
   │
3. 提取关键信息
   ├── 品牌信息: Logo, 品牌色, 字体
   ├── 产品信息: 外观, 特点, 材质
   ├── 设计规范: 布局, 风格, 元素
   └── 视觉参考: 图片, 配色, 构图
   │
4. 整合到上下文
   │
5. 转化为提示词元素
```

### 提取策略

| 网页类型 | 重点提取 | 忽略 |
|---------|---------|------|
| 品牌官网 | 品牌色、Logo、字体、产品图 | 导航栏、页脚、广告 |
| 产品页面 | 产品外观、材质、尺寸、配色 | 价格、库存、评论 |
| 设计作品集 | 风格、构图、配色、技法 | 个人简介、联系方式 |
| 参考文章 | 关键数据、专业术语、方法论 | 广告、侧边栏 |

### 信息验证

提取后必须验证：
- □ 信息是否与当前任务相关？
- □ 信息是否准确？（交叉验证）
- □ 信息是否最新？（检查时效性）
- □ 是否有冲突信息需要解决？

## retrieve_skills 详细流程

### 功能

检索设计技能指导，获取设计流程、最佳实践、专业指导

### 可用技能详细说明

| 技能ID | 名称 | 适用场景 | 输出内容 |
|--------|------|---------|---------|
| amazon_product_image | 亚马逊产品图 | 电商产品主图、A+页面 | 布局规范、拍摄角度、背景要求、尺寸标准 |
| instagram_post | 社交媒体帖子 | Instagram帖子、Stories | 尺寸规格、视觉风格、文字排版、标签策略 |
| branding_logo | 品牌Logo | Logo设计、品牌标识 | 设计原则、变体规范、配色方案、应用场景 |
| brochure_design | 宣传册设计 | 折页、画册、手册 | 版式规范、图文比例、印刷要求、折叠方式 |
| storyboard_design | 故事板设计 | 视频分镜、动画脚本 | 镜头语言、构图规范、标注方式、序列组织 |
| carousel_design | 轮播图设计 | 多图轮播、产品展示 | 图片数量、过渡效果、信息层次、导航设计 |

### 使用流程

```
1. 分析任务类型
   │
2. 匹配可用技能
   │
3. 调用 retrieve_skills(skill_id)
   │
4. 获取技能指导
   ├── 设计流程: 步骤和顺序
   ├── 最佳实践: 应该做什么
   ├── 常见错误: 不应该做什么
   └── 搜索策略: 如何搜索参考
   │
5. 应用到当前任务
   │
6. 按技能指导执行
```

### 关键规则

- **不要同时调用 retrieve_skills 和 search_image**
- 先检索技能，技能中包含搜索策略
- 按技能指导进行后续搜索
- 技能指导优先于个人判断

## 检索结果整合实战案例

### 案例：品牌产品海报设计

**用户需求**："帮我设计一个科技产品的海报，品牌是XX科技"

**Step 1：知识盲区识别**

```
已知: 海报设计、科技风格
未知: XX科技的品牌规范、产品外观
需要检索: 品牌信息 + 产品信息 + 设计参考
```

**Step 2：检索执行**

```
第1步: search_web("XX科技 品牌")
  → 获取品牌基本信息

第2步: search_website_content(品牌官网URL)
  → 提取品牌色、Logo、字体、产品图

第3步: retrieve_skills("instagram_post")
  → 获取社交媒体帖子设计指导

第4步: search_image("tech product poster", biz_type=8)
  → 获取设计参考图
```

**Step 3：信息整合**

```
品牌信息:
  品牌色: #0066CC (科技蓝)
  辅助色: #FFFFFF (白色)
  字体: Helvetica Neue
  风格: 极简、科技感

产品信息:
  外观: 圆柱形, 白色, 顶部LED灯
  特点: 智能语音控制
  尺寸: 10cm x 10cm x 15cm

设计指导:
  尺寸: 1080x1080 (Instagram)
  风格: 极简科技
  布局: 产品居中, 品牌色背景
  文字: 品牌名+产品名+Slogan
```

**Step 4：转化为提示词**

```
Tech product poster, minimalist design,
XX Tech brand color #0066CC background,
white cylindrical smart speaker with LED on top,
centered product placement,
clean typography, Helvetica Neue,
brand logo at top, product name below,
slogan at bottom,
professional product photography,
studio lighting, soft shadows,
1080x1080 format
```

**Step 5：应用检索结果**

使用品牌色作为背景色，产品外观描述融入提示词，遵循设计指导的布局规范。

## 检索质量检查清单

每次完成知识检索后，检查以下项目：

- □ 是否识别了知识盲区？
- □ 是否选择了正确的检索工具？
- □ 检索结果是否与任务相关？
- □ 信息是否经过验证？
- □ 是否整合到3层上下文模型？
- □ 是否转化为可用的提示词元素？
- □ 是否遵循了技能指导？
- □ 是否有遗漏的关键信息？
- □ 检索顺序是否正确（先技能后搜索）？
- □ 是否避免了同时调用 retrieve_skills 和 search_image？
