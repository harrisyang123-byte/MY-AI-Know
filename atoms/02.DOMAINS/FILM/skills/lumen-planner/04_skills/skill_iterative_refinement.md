# Skill 6: Iterative Refinement (迭代优化)

## 核心理念

### 什么是 Iterative Refinement？

通过反馈循环，逐步优化，直到达到目标。

**不是**：
- "一次就完美"
- "不行就重来"
- "随机试错"

**而是**：
- 分析反馈
- 识别问题
- 针对性优化
- 逐步逼近目标

### 为什么重要？

现实：
- 第一次生成很少完美
- 用户需求可能不明确
- 工具有随机性
- 复杂任务需要多次调整

解决方案：
- 迭代优化
- 从反馈中学习
- 逐步提升质量
- 最终达到满意结果

---

## 核心循环

```
    ┌─────────────┐
    │   Generate  │ 生成
    └──────┬──────┘
           │
           ▼
    ┌─────────────┐
    │   Evaluate  │ 评估
    └──────┬──────┘
           │
           ▼
    ┌─────────────┐
    │   Analyze   │ 分析
    └──────┬──────┘
           │
           ▼
    ┌─────────────┐
    │   Refine    │ 优化
    └──────┬──────┘
           │
           └──────→ 回到 Generate
           
  直到: 达到目标 或 收敛
```

---

## 迭代循环详解

### 第一步：Generate (生成)

**目标**：快速生成第一版，建立基础，获取反馈

**原则**：
- 不追求完美
- 关注核心需求
- 快速迭代

**策略**：
- 先满足基本布局
- 先实现核心元素
- 不纠结细节

---

### 第二步：Evaluate (评估)

**目标**：识别问题，发现差距，收集反馈

**评估维度**：

1. **用户反馈**
   - 用户说了什么？
   - 满意还是不满意？
   - 具体哪里不满意？

2. **需求匹配**
   - 是否符合原始需求？
   - 哪些需求满足了？
   - 哪些需求没满足？

3. **质量评估**
   - 视觉质量如何？
   - 技术质量如何？
   - 细节是否到位？

4. **可优化空间**
   - 哪些可以改进？
   - 改进的优先级？
   - 改进的可行性？

---

### 第三步：Analyze (分析)

**目标**：找到根本原因，确定优化方向，制定优化策略

**问题分类框架**：

| 类型 | 描述 | 解决方向 |
|------|------|---------|
| 类型A: 提示词问题 | 描述不够清晰/重点不够突出/细节不够具体 | 增强提示词 |
| 类型B: 参数问题 | 尺寸不合适/质量设置不当/比例不对 | 调整参数 |
| 类型C: 工具问题 | 工具选择不当/工具能力限制 | 换工具 |
| 类型D: 需求问题 | 需求不明确/需求有冲突 | 澄清需求 |

**根因分析方法**：

连续问"为什么"直到找到根本原因：

```
问题: 产品太小
  → 为什么? 提示词中没有强调产品大小
  → 根因: 提示词权重不够
  → 策略: 在提示词中明确 "large product, prominent"
```

---

### 第四步：Refine (优化)

**目标**：应用优化策略，生成新版本，解决已识别问题

**优化方法**：

**方法1：提示词优化**

```
原提示词:
  "Product poster, modern style, clean background"

优化后:
  "Product poster, modern style, clean background,
   LARGE PRODUCT in center (占画面60%),
   BOLD TITLE at top (eye-catching, prominent),
   clear visual hierarchy: product > title > details,
   professional, high-end"
```

**方法2：参数优化**

```
原参数:
  size: "2048x2048"
  quality: "medium"

优化后:
  size: "2048x2560" (竖版更适合海报)
  quality: "high" (用户要求高质量)
```

**方法3：工具切换**

```
原工具: nano_banana_2
问题: 控制力不够
优化: 切换到 gpt_image_2
```

**方法4：增量编辑**

```
策略: 基于第一版进行编辑
实现:
  → 使用第一版作为 base_image
  → 在提示词中明确保留什么、改变什么
  → "Keep the overall layout and style,
     but make the product LARGER and
     make the title more PROMINENT"
```

**优化原则**：
- 针对性优化：只改需要改的，不要全盘推翻
- 增量优化：一次改一点，不要一次改太多
- 保留优点：明确保留好的部分，只改不好的部分
- 可追溯：记录每次改动，知道改了什么

## 3种迭代模式

### 模式1：快速迭代 (Rapid Iteration)

**特点**：快速生成多个版本，快速试错，探索不同方向

**适用场景**：
- 需求不明确
- 探索阶段
- 寻找最佳方向

**流程**：
```
V1 → 评估 → V2 → 评估 → V3 → 评估 → 选最佳

每个版本:
  → 不同的风格/构图/配色
  → 快速生成
  → 用户选择
```

**示例**：
```
任务: 设计Logo

V1: 极简风格
V2: 科技风格
V3: 传统风格

用户选择: V2 (科技风格)

然后基于V2继续优化
```

---

### 模式2：增量优化 (Incremental Refinement)

**特点**：基于现有版本，逐步改进，保留优点

**适用场景**：
- 方向已确定
- 需要细化
- 逐步完善

**流程**：
```
V1 → 分析问题 → V2 (改进A) → V3 (改进B) → V4 (改进C)

每个版本:
  → 解决一个具体问题
  → 保留之前的优点
  → 逐步逼近目标
```

**示例**：
```
任务: 产品海报

V1: 基础版本
V2: 放大产品 (解决产品太小)
V3: 增强标题 (解决标题不突出)
V4: 优化配色 (提升视觉效果)

每次只改一个方面
```

---

### 模式3：分支探索 (Branch Exploration)

**特点**：从某个版本分支，探索不同优化方向，对比选择

**适用场景**：
- 有多个优化方向
- 不确定哪个更好
- 需要对比

**流程**：
```
V1 → 分支A (方向1) → V2a
   → 分支B (方向2) → V2b
   → 分支C (方向3) → V2c

对比 V2a, V2b, V2c → 选最佳 → 继续优化
```

**示例**：
```
任务: 风景画

V1: 基础风景

分支A: 增强光影效果
分支B: 增强色彩饱和度
分支C: 增强细节质感

对比后选择: 分支A (光影效果最好)

基于分支A继续优化
```

## 收敛策略

### 何时停止迭代？

**停止条件**：

1. **目标达成**
   - 用户满意
   - 需求全部满足
   - 质量达标

2. **收益递减**
   - 改进越来越小
   - 投入产出比下降
   - 接近工具极限

3. **资源限制**
   - 时间限制
   - 成本限制
   - 迭代次数限制

### 判断标准

**标准1：用户反馈**

积极信号：
- "很好"、"满意"、"可以了"、"没问题"
→ 可以停止

**标准2：需求覆盖**

检查清单：
- □ 核心需求满足？
- □ 次要需求满足？
- □ 质量要求达标？
- □ 没有明显问题？
全部 ✓ → 可以停止

**标准3：改进幅度**

对比：
- V(n) vs V(n-1): 改进 < 5%
- V(n-1) vs V(n-2): 改进 < 5%
→ 接近收敛，可以停止

### 停止策略

1. **主动确认**："这个版本怎么样？还需要调整吗？"
2. **提供选项**："我可以继续优化，或者就用这个版本，你觉得呢？"
3. **说明限制**："这个已经接近工具的极限了，继续优化可能改进不大"

## 错误模式识别与解决

### 常见错误模式

| 错误类型 | 根因 | 解决方案 |
|---------|------|---------|
| 门窗纸发光 | 提示词未明确光源位置 | 正向："only door crack emits light"；负向："door paper glowing" |
| 现代元素混入 | 时代特征描述不够强 | 加强时代描述 + 负向提示词 |
| 光线过亮 | 未量化光线强度 | 量化光线比例（烛火95%，月光5%） |
| 空间错误 | 未做空间推理 | 先走5步空间推理流程 |
| 人物不一致 | 缺少参考图 | 使用链式参考策略 |
| 风格偏离 | 风格描述不够具体 | 添加参考作品 + 详细风格描述 |

### 预防性检查

每次生成前检查：
- □ 空间关系是否合理？
- □ 光源位置是否明确？
- □ 时代特征是否准确？
- □ 负向提示词是否包含已知错误？
- □ 参考图是否正确使用？

## 版本管理规范

格式：`[场景名]-[版本号]-[特征]`

示例：
- 西向东拍-v1-初版


## 完整实战案例

### 案例1：产品海报迭代（增量优化模式）

**任务**：设计一张产品海报

**V1: 初版生成**

用户需求："设计一张产品海报，现代风格"

策略：快速生成第一版，满足基本需求，不追求完美

工具：generate_image_gpt_image_2

提示词："Product poster, modern style, clean background, professional"

参数：size: "2048x2560", quality: "medium"

结果：基本布局合理，但有改进空间

**V1 评估**

用户反馈："整体不错，但产品太小了，标题也不够突出"

问题识别：
- ✗ 产品尺寸：太小
- ✗ 标题突出度：不够
- ✓ 整体风格：满意
- ✓ 配色：满意

**V1 分析**

| 问题 | 分类 | 根因 | 策略 |
|------|------|------|------|
| 产品太小 | 提示词问题 | 没有强调产品大小 | 明确 "LARGE PRODUCT" |
| 标题不突出 | 提示词问题 | 标题权重不够 | 强调 "BOLD TITLE" |

**V2: 第一次优化**

策略：基于V1优化，解决产品大小和标题问题，保留风格和配色

提示词：
```
Product poster, modern style, clean background, professional,
LARGE PRODUCT prominently displayed (占画面60%),
BOLD TITLE at top (eye-catching, prominent),
keep the overall style and color scheme
```

结果：产品明显变大，标题更突出，保留了原有风格

**V2 评估**

用户反馈："好多了！但能不能让背景更有科技感？"

问题识别：✓ 产品尺寸：满意 ✓ 标题突出度：满意 ✗ 背景：希望更有科技感

**V3: 第二次优化**

策略：增强背景科技感，保留产品和标题

提示词：
```
Product poster, modern style,
LARGE PRODUCT prominently displayed (占画面60%),
BOLD TITLE at top (eye-catching, prominent),
TECH BACKGROUND with subtle geometric patterns,
gradient lighting, futuristic, high-tech atmosphere,
professional, premium
```

**V3 评估**

用户反馈："完美！就用这个！"

结果：✓ 所有需求满足 ✓ 用户满意 ✓ 迭代结束

**总结**：迭代3次，增量优化模式。关键成功因素：每次只改一个方面，保留已满意的部分，基于用户反馈优化。

### 案例2：Logo设计迭代（快速迭代 + 分支探索模式）

**任务**：设计一个科技公司Logo

**V1: 多方向探索**

生成3个版本：
- V1a：极简风格 — "minimalist, simple geometric shapes, monochrome"
- V1b：科技风格 — "tech style, circuit patterns, blue gradient, modern"
- V1c：流动风格 — "flowing lines, dynamic, fluid shapes, gradient colors"

用户选择：V1b（科技风格），但希望更简洁

**V2: 基于V1b简化**

提示词："Tech logo, tech style, SIMPLIFIED design, clean lines, blue gradient, modern, digital, less complex, more recognizable"

用户反馈："很好！能不能试试不同的配色？"

**V3: 配色探索（分支探索）**

生成3个配色版本：V3a蓝紫渐变、V3b绿色科技、V3c橙色活力

用户选择：V3a（蓝紫渐变）

**总结**：3轮迭代，快速迭代 + 分支探索。关键：先确定大方向，再优化细节，给用户选择权。

## 高级技巧

### 反馈分析技巧

**识别反馈类型**：

| 类型 | 示例 | 应对 |
|------|------|------|
| 明确反馈 | "产品太小了" | 直接优化 |
| 模糊反馈 | "感觉不太对" | 追问具体哪里不对 |
| 情感反馈 | "不喜欢" | 追问为什么不喜欢 |
| 建议反馈 | "能不能加个..." | 明确的改进建议 |

**追问策略**：
- 模糊反馈 → 追问具体："具体是哪个部分不太对？是配色、布局还是风格？"
- 情感反馈 → 追问原因："能说说具体不喜欢哪里吗？"
- 矛盾反馈 → 澄清优先级："简洁和信息量有时会冲突，哪个更重要？"

### 提示词优化技巧

**权重强调**：大写关键词 + 重复关键词 + 使用强调词

```
原: "product poster with large product"
优: "product poster with LARGE PRODUCT (prominent, 占画面60%)"
```

**保留指令**：明确说明保留什么

```
"Keep the overall style and color scheme, but make the product LARGER"
```

**对比描述**：说明要改什么 + 不要改什么

```
"Change: make product larger, title more prominent
 Keep: color scheme, overall layout, background style"
```

**层次描述**：明确视觉层次

```
"Visual hierarchy: 1. Product (most prominent, 60% of frame)
 2. Title (bold, eye-catching) 3. Details (subtle, supporting)"
```

### 迭代效率优化

1. **批量生成**：不确定方向时，一次生成多个版本让用户选择
2. **主动建议**：发现潜在问题时，主动指出并提供优化建议
3. **设定检查点**：复杂任务分阶段确认，避免大返工
4. **记录迭代历史**：记录每次改动和用户反馈，避免重复错误

## 系统化错误模式识别与解决方案库

### 错误分类体系

```
错误类型
├── 空间错误 (Spatial Errors)
│   ├── 透视错误
│   ├── 比例错误
│   ├── 位置错误
│   └── 可见性错误
├── 光线错误 (Lighting Errors)
│   ├── 光源位置错误
│   ├── 光线强度错误
│   ├── 色温错误
│   └── 阴影方向错误
├── 元素错误 (Element Errors)
│   ├── 时代错误
│   ├── 材质错误
│   ├── 细节缺失
│   └── 多余元素
├── 风格错误 (Style Errors)
│   ├── 风格偏离
│   ├── 一致性丢失
│   └── 质量不足
└── 人物错误 (Character Errors)
    ├── 外貌不一致
    ├── 姿态不合理
    └── 服饰错误
```

### 详细错误模式与解决方案

#### 空间错误

| 错误模式 | 识别方法 | 根因 | 解决方案 | 预防措施 |
|---------|---------|------|---------|---------|
| 摄影机后方物体可见 | 画平面图验证视角 | 未做空间推理 | 先走5步空间推理，画平面图 | 每次生成前画平面图 |
| 家具比例失调 | 对比人体尺度 | 未标注具体尺寸 | 所有元素标注具体米数 | 使用标准家具尺寸参考 |
| 门窗位置矛盾 | 检查视角与门窗关系 | 视角定义模糊 | 使用绝对方向(东西南北) | 视角定义必须含方向 |
| 空间过于拥挤/空旷 | 计算家具占地面积比 | 未考虑功能分区 | 按40-60%填充率规划 | 先规划功能分区再放家具 |
| 墙面/地面/天花板消失 | 检查3个面是否都可见 | 视角定义不完整 | 明确相机位置和朝向 | 每次验证3面可见性 |

#### 光线错误

| 错误模式 | 识别方法 | 根因 | 解决方案 | 预防措施 |
|---------|---------|------|---------|---------|
| 门窗纸/墙面自发光 | 检查非光源物体是否发光 | 未明确光源位置 | 正向："only door crack emits light"；负向："door paper glowing" | 每次量化光源位置和强度 |
| 整体过亮/过暗 | 对比场景时间判断 | 未量化光线强度 | 量化光线比例(烛火95%, 月光5%) | 多光源必须量化比例 |
| 色温不匹配 | 检查光源色温是否符合时代 | 未指定色温 | 指定色温K值(烛光2800K, 日光5500K) | 使用色温参考表 |
| 阴影方向错误 | 从光源位置推导阴影方向 | 光源与阴影不一致 | 从光源位置推导阴影方向 | 光源定义后立即验证阴影 |
| 多光源冲突 | 检查光源之间是否矛盾 | 多光源未量化主次 | 明确主光/补光/轮廓光比例 | 多光源必须量化比例总和=100% |

#### 元素错误

| 错误模式 | 识别方法 | 根因 | 解决方案 | 预防措施 |
|---------|---------|------|---------|---------|
| 现代元素混入历史场景 | 检查每个元素是否符合时代 | 时代特征描述不够强 | 加强时代描述 + 负向提示词 | 按时代检查每个元素 |
| 材质不符合时代 | 检查材质是否历史准确 | 未研究时代材质 | 参考历史资料确认材质 | 使用时代材质参考表 |
| 关键细节缺失 | 对比需求检查元素完整性 | 元素描述不够详细 | 每个元素6要素(名称+位置+尺寸+材质+样式+状态) | 使用元素描述模板 |
| 多余元素出现 | 检查是否有未提及的元素 | AI自由发挥过度 | 负向提示词排除 + 更精确的描述 | 限制性描述 + 负向提示词 |

#### 风格错误

| 错误模式 | 识别方法 | 根因 | 解决方案 | 预防措施 |
|---------|---------|------|---------|---------|
| 风格偏离参考 | 对比参考图/描述 | 风格描述不够具体 | 添加参考作品 + 详细风格描述 | 每次明确3个风格关键词 |
| 多图风格不一致 | 对比系列图像 | 缺少风格锚定 | 第一张图确立基准，后续链式参考 | 使用参考图保持一致性 |
| 质量不达标 | 检查细节清晰度 | 参数配置不当 | 调整quality参数 + 增强细节描述 | 按需求选择quality级别 |

#### 人物错误

| 错误模式 | 识别方法 | 根因 | 解决方案 | 预防措施 |
|---------|---------|------|---------|---------|
| 同一人物外貌变化 | 对比多张图中人物 | 缺少参考图 | 使用链式参考策略 | 第一次详细描述，后续用参考图 |
| 姿态不合理 | 检查人体工学 | 姿态描述不够具体 | 详细描述姿态+动作+重心 | 使用具体姿态描述 |
| 服饰不符合时代/身份 | 检查服饰与角色匹配 | 服饰描述不够详细 | 详细描述服饰(材质+颜色+样式+配饰) | 按时代和身份描述服饰 |

### 错误模式快速诊断流程

```
发现问题
  │
  ├─ 空间不对? → 空间错误 → 重新做空间推理
  │
  ├─ 光线不对? → 光线错误 → 量化光源位置和强度
  │
  ├─ 元素不对? → 元素错误 → 检查时代/材质/细节
  │
  ├─ 风格不对? → 风格错误 → 加强风格描述和参考
  │
  └─ 人物不对? → 人物错误 → 详细描述+参考图
```

### 解决方案库使用方法

1. **识别错误类型**：根据上述分类确定错误属于哪一类
2. **查找根因**：从错误模式表中找到根因
3. **应用解决方案**：按照解决方案执行
4. **添加负向提示词**：将错误转化为负向提示词，防止再犯
5. **记录到经验库**：将错误-解决方案对记录下来，供未来参考

## 质量评估体系

### 评估维度

| 维度 | 权重 | 评估标准 | 评分 |
|------|------|---------|------|
| 需求符合度 | 30% | 是否满足用户所有明确需求 | 1-5 |
| 空间准确性 | 20% | 空间关系是否正确 | 1-5 |
| 光线合理性 | 15% | 光线设置是否合理 | 1-5 |
| 细节完整度 | 15% | 所有元素是否完整呈现 | 1-5 |
| 风格一致性 | 10% | 风格是否与需求一致 | 1-5 |
| 技术质量 | 10% | 分辨率、清晰度等 | 1-5 |

### 评分标准

- **5分**：完美，无需修改
- **4分**：很好，微小改进空间
- **3分**：及格，有明显改进空间
- **2分**：不足，需要较大修改
- **1分**：失败，需要重新生成

### 评估流程

1. 按维度逐一评分
2. 计算加权总分
3. 总分 ≥ 4.0 → 可以交付
4. 总分 3.0-3.9 → 需要优化
5. 总分 < 3.0 → 需要重新生成

### A/B测试策略

当不确定哪个方向更好时：

1. **生成2-3个变体**：只改变一个变量（如光线、配色、构图）
2. **对比评估**：按质量评估体系打分
3. **用户选择**：让用户选择偏好
4. **基于选择优化**：在用户选择的方向上继续迭代

## 自动化改进建议

### 目的

主动提供优化建议和后续方向，不等用户问才回答

### 建议类型

| 类型 | 示例 | 触发条件 |
|------|------|---------|
| 技术优化 | "当前分辨率2K，是否需要4K高清版？" | 生成完成后 |
| 内容扩展 | "已完成北向南和西向东，是否需要其他角度？" | 多角度项目 |
| 细节特写 | "是否需要生成细节特写（书桌、门、窗）？" | 复杂场景 |
| 应用场景 | "这个场景可以制作成视频吗？" | 场景完成后 |
| 时间变体 | "是否需要生成不同时间（白天/黄昏）版本？" | 单一时间场景 |
| 风格变化 | "是否尝试更写实/更艺术化的风格？" | 风格确定后 |
| 系列扩展 | "是否需要生成配套的社交媒体版本？" | 海报/品牌项目 |

### 建议原则

1. **每次生成后最多提供2-3个建议**：不要过多，避免用户选择困难
2. **建议必须具体可行**：不要泛泛而谈，要给出具体操作
3. **建议基于上下文**：根据项目类型和用户偏好定制
4. **不强制执行**：只是建议，用户可以选择忽略
5. **记录用户选择**：用户接受/拒绝的建议都要记录，用于学习偏好

### 建议生成流程

```
生成完成
  │
  ├─ 分析当前成果
  │   ├── 已完成什么？
  │   ├── 质量如何？
  │   └── 还有什么可以改进？
  │
  ├─ 识别改进空间
  │   ├── 技术层面（分辨率/质量/细节）
  │   ├── 内容层面（角度/细节/变体）
  │   └── 应用层面（用途/平台/系列）
  │
  ├─ 生成2-3个建议
  │   ├── 每个建议包含：描述 + 预期效果 + 操作方式
  │   └── 按相关性排序
  │
  └─ 呈现给用户
      ├── 简洁明了
      └── 不强制
```

### 建议示例

**场景1：完成一张清代办公室图像后**

```
✓ 当前版本已完成

💡 后续建议:
  1. 生成其他角度（东向西/南向北），保持同一房间一致性
  2. 生成细节特写（桌面文件/门框雕花/窗纸透光效果）
  3. 生成不同时间版本（白天/黄昏），展示同一空间不同氛围
```

```
  3. 尝试不同背景色变体，A/B测试哪个更吸引
```
