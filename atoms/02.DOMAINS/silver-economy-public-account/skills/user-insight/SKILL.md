# Skill: 用户洞察

## 概述

银发经济有两个核心受众，必须先搞清楚在对谁说话：

- **受众A：银发群体的子女**（30-50岁）— 焦虑的中间代，夹在父母和孩子之间
- **受众B：银发群体本身**（55岁+）— 有价值焦虑、情感需求、健康关注

两个受众的需求完全不同，内容策略也不同。

## 分析框架

### 第一步：受众确认

明确当前内容面向哪个受众，或两者兼顾。

**受众A（子女）的核心焦虑**：
- 父母不理解自己的生活方式和选择
- 不知道如何与父母沟通而不引发冲突
- 担心父母的健康和晚年生活质量
- 被催婚催育的压力

**受众B（银发群体）的核心焦虑**：
- 退休后的价值感和存在感
- 与子女的代际隔阂
- 健康焦虑（疾病、衰老）
- 被骗、被忽视的恐惧
- 跟不上时代的失落感

### 第二步：需求三层模型

对每个痛点进行三层拆解：

```
表层需求（用户说想要什么）
    ↓
深层需求（用户真正想解决什么）
    ↓
终极需求（背后的情感/价值诉求）
```

**示例**：
- 表层：想知道如何跟父母说不催婚
- 深层：想在不伤感情的前提下维护自己的边界
- 终极：被父母理解和尊重，同时保持亲密关系

### 第三步：代际冲突地图

银发经济内容的核心张力来自代际冲突，建立冲突地图：

| 冲突场景 | 子女视角 | 父母视角 | 内容切入点 |
|---------|---------|---------|-----------|
| 催婚催育 | 被控制 | 为你好 | 理解背后的恐惧 |
| 生活方式 | 不被理解 | 担心孩子 | 翻译彼此的语言 |
| 金钱观念 | 价值观不同 | 节俭是美德 | 时代背景差异 |
| 健康问题 | 父母不听劝 | 不想麻烦孩子 | 建立信任的方式 |

### 第四步：情感需求建模

银发内容的核心是情感共鸣，不是信息传递。建立情感需求模型：

**高价值情感触点**（按共鸣强度排序）：
1. 被理解的渴望（"终于有人说出我的感受"）
2. 关系修复的希望（"原来可以这样沟通"）
3. 价值感的确认（"我的选择是对的"）
4. 焦虑的缓解（"这个问题有解"）

### 第五步：用户画像更新

基于分析结果，更新用户画像文档：

```markdown
## 核心用户画像

### 画像A：焦虑的中间代
- 年龄：30-45岁
- 特征：大城市工作，父母在老家或同城
- 核心痛点：代际沟通障碍，被催婚催育
- 内容偏好：实用的沟通技巧，情感共鸣
- 触达场景：通勤、睡前刷手机

### 画像B：寻找价值感的银发族
- 年龄：55-70岁
- 特征：退休或即将退休，子女已成年
- 核心痛点：价值感缺失，与子女疏远
- 内容偏好：被理解的感受，正向引导
- 触达场景：早晨、饭后休息时间
```

## 输出格式

每次洞察分析输出：
1. 本次分析的受众焦点
2. 识别到的3-5个核心痛点（含三层需求拆解）
3. 更新的代际冲突地图条目
4. 对内容策略的直接建议
