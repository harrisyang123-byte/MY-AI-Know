# Skill: 内容策略

## 概述

内容策略不是拍脑袋决定写什么，而是在用户需求、竞品空白、自身优势三者的交叉点上找到最优解。

## 核心定位（已确定，不需要每次重新讨论）

**定位**：产品经理系统化解决银发群体的真实需求  
**差异化**：不是情绪宣泄，而是给出系统性解决方案  
**口号**：不是教科技，而是解心结  
**不做什么**：纯情绪煽动、无解决方案的批判、健康养生科普（竞争过度）

## 分析框架

### 第一步：选题优先级矩阵

用两个维度评估每个选题：

```
高用户需求
    │
    │  ★★★★          ★★★★★
    │  高需求低竞争    高需求高竞争
    │  （优先做）      （差异化切入）
    │
    │  ★★             ★★★
    │  低需求低竞争    低需求高竞争
    │  （暂不做）      （不做）
    └─────────────────────────────
低竞争                        高竞争
```

对每个候选选题打分（1-5分），输出优先级排序。

### 第二步：内容类型规划

基于定位，确定核心内容类型：

**类型A：代际沟通解决方案**（主打，占40%）
- 具体场景 + 系统性解法
- 示例：「催婚背后的恐惧：用产品思维理解父母的需求」

**类型B：银发价值重建**（占25%）
- 帮助银发群体找到退休后的价值感
- 示例：「退休不是终点：如何用30年经验创造新价值」

**类型C：代际文化翻译**（占20%）
- 解读两代人之间的文化和价值观差异
- 示例：「为什么父母总觉得你不孝：两代人的孝道定义完全不同」

**类型D：数据洞察**（占15%）
- 用数据说话，建立专业感
- 示例：「3.2亿老年人最怕什么：银发焦虑的真实数据」

### 第三步：差异化执行原则

每篇文章必须体现产品经理视角：

1. **问题定义清晰**：不是"父母不好"，而是"这个系统出了什么问题"
2. **根因分析**：找到行为背后的真实动机
3. **解决方案具体**：不是"多沟通"，而是"用这3步建立沟通框架"
4. **可验证**：给出可以立刻尝试的行动

### 第四步：发布节奏规划

**频率建议**：每周3篇（质量优先于数量）
- 周一：代际沟通解决方案（工作日开始，情绪需求高）
- 周四：银发价值重建或文化翻译
- 周六：数据洞察或轻量内容（周末阅读习惯不同）

**A/B测试计划**：
每月选1个变量进行测试，记录数据：
- 标题风格（警示型 vs 解决方案型）
- 文章长度（1000字 vs 2000字）
- 开头方式（场景切入 vs 数据切入）

### 第五步：内容日历

输出未来2-4周的内容日历：

```markdown
| 日期 | 类型 | 选题 | 目标受众 | 核心假设 |
|------|------|------|---------|---------|
| ... | ... | ... | ... | ... |
```

## 输出格式

每次策略制定输出：
1. 选题优先级矩阵（本批次候选选题评分）
2. 推荐的内容类型组合
3. 未来2-4周内容日历
4. 本阶段要验证的核心假设
