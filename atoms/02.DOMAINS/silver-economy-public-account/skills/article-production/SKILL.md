# Skill: 文章生产

## 概述

每篇文章都是一个产品。有目标用户、有核心价值主张、有结构设计、有情绪节奏。不是写完就发，而是经过校验的可发布版本。

## 生产流程

### 第一步：选题确认

在动笔之前，回答这三个问题：

1. **这篇文章解决谁的什么问题？**（具体到人群和场景）
2. **读完之后，读者会有什么不同？**（认知改变 or 情绪释放 or 行动改变）
3. **为什么是我们来写这篇？**（产品经理视角能带来什么独特价值）

如果三个问题答不清楚，选题需要重新确认。

### 第二步：结构设计

根据内容类型选择结构模板：

**模板A：解决方案型**（代际沟通类）
```
① 场景还原（100字）：一个读者立刻能对号入座的具体场景
② 问题定义（200字）：这不是个人问题，而是系统问题
③ 根因分析（300字）：为什么会这样？背后的真实动机是什么？
④ 解决方案（400字）：3步或3个原则，具体可操作
⑤ 情感落点（100字）：不是说教，而是理解和希望
```

**模板B：价值重建型**（银发价值类）
```
① 共情开场（150字）：说出银发群体的内心感受
② 现实确认（200字）：这种感受是真实的，不是矫情
③ 重新定义（300字）：换一个框架看待这件事
④ 行动路径（300字）：具体可以怎么做
⑤ 愿景收尾（100字）：描绘改变后的状态
```

**模板C：文化翻译型**（代际文化类）
```
① 冲突场景（100字）：两代人的典型冲突
② 子女视角（200字）：子女为什么这么想
③ 父母视角（200字）：父母为什么那么做
④ 文化根源（300字）：时代背景和价值观差异
⑤ 翻译建议（200字）：如何在两种语言之间建立桥梁
```

### 第三步：标题创作

参考 `references/title-formulas.md`，生成3-5个候选标题：

**评估标准**：
- 目标用户看到标题，会不会立刻想点开？
- 标题是否准确传递了文章的核心价值？
- 是否避免了标题党（承诺了文章没有兑现的内容）？

### 第四步：初稿生成

按选定的结构模板生成完整初稿。

**写作原则**：
- 用"你"而不是"他们"，直接对话读者
- 每段不超过150字，银发受众阅读习惯
- 避免学术语言，用生活化表达
- 产品经理视角体现在：系统性分析、根因挖掘、具体解法

### 第五步：情绪校验

逐段检查情绪节奏：

```
开头：是否制造了足够的代入感？（1-5分）
中段：情绪是否有起伏？还是平铺直叙？（1-5分）
结尾：情绪落点是希望/力量，还是焦虑/无力？（必须是前者）
整体：是否有一个情绪最高点？在哪里？
```

如果情绪校验不通过，返回第四步修改。

### 第六步：发布版本输出

```markdown
【标题】（最终选定）

【正文】

【配图建议】
- 封面图风格：...
- 文中插图建议：...

【发布信息】
- 目标受众：...
- 内容类型：...
- 核心假设：...（这篇文章验证什么假设）
- 预期指标：阅读量X+，互动率X%+
```

## 质量标准

每篇文章发布前必须满足：
- [ ] 选题三问已回答
- [ ] 结构模板已选定并执行
- [ ] 至少3个候选标题
- [ ] 情绪校验通过（结尾是正向落点）
- [ ] 产品经理视角体现（有根因分析或系统性解法）
