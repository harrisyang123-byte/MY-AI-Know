# Skill 1.1: 多模态内容分析

## 核心思想

用户输入通常包含多种信息模态（文本、图片、历史上下文），需要系统化提取每一层信息，不遗漏任何关键信号。

## 四层分析模型

### 第一层：显性信息提取

| 类别 | 提取内容 | 示例 |
|------|----------|------|
| 名词 | 场景、物体、人物 | "办公室"、"书桌"、"张十三" |
| 动词 | 动作、状态 | "坐在"、"推开" |
| 形容词 | 描述性词汇 | "昏暗的"、"深色的" |
| 数量词 | 具体数值 | "4扇窗户"、"3-5cm窄缝" |

约束识别：
- 必须有："一定要"、"必须"、"需要"
- 不能有："不要"、"避免"、"去掉"
- 偏好："最好"、"希望"、"倾向于"

技术参数提取：
- 尺寸：1920x1080、16:9
- 数量：3张、5个角度
- 时间：截止日期、优先级

### 第二层：隐性意图推断

用途推断：
- "用于社交媒体" → 需要吸引眼球，色彩鲜艳
- "用于打印" → 需要高分辨率，CMYK色彩
- "用于演示" → 需要清晰易读，简洁明了

受众推断：
- "给客户看" → 专业、精致
- "自己用" → 可以快速迭代
- "给团队参考" → 需要标注说明

风格推断：
- "历史场景" → 考据准确性
- "科幻场景" → 想象力和未来感
- "商业场景" → 吸引力和转化率

缺失信息识别：
- 识别缺失的关键信息
- 判断是否需要询问
- 或基于经验做合理假设

### 第三层：情感与态度

紧迫度：
- 紧急："尽快"、"马上"、"今天"
- 不急："有空的时候"、"不着急"

满意度信号：
- 满意："很好"、"完美"、"就是这样"
- 不满意："不对"、"不是这样"、"差远了"
- 部分满意："还可以，但是..."

开放度：
- 开放："随便"、"你看着办"、"给我惊喜"
- 严格："必须"、"一定要"、"不能有任何偏差"

### 第四层：对话历史关联

- 是否是延续性任务？
- 是否需要保持一致性？
- 是否是修正之前的错误？
- 用户偏好的风格是什么？
- 用户的质量标准是什么？

## 实战案例

**用户输入**："空间不对...从西向东拍，你认为还能看到书架吗？书架在西墙。"

| 层级 | 分析结果 |
|------|----------|
| 显性 | 关键词：空间、西向东拍、书架、西墙；约束：不能看到书架；问题：空间关系错误 |
| 隐性 | 这是对上一次生成的反馈；用户理解空间关系，我理解错了；用户在纠正错误 |
| 情感 | 紧迫度：中等；满意度：不满意；语气：教育性的 |
| 历史 | 上次错误：提示词中包含了"西墙书柜"；我未理解"摄影机在西墙"的含义 |

**响应策略**：
1. 道歉并承认错误
2. 解释正确的空间关系
3. 说明修正方案（删除书柜描述）
4. 重新生成
5. 确认是否解决问题
