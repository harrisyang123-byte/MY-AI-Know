# Skill 1.3: 空间关系推理

## 核心思想

3D 空间理解是最容易出错的地方。需要建立严格的空间推理系统。

## 坐标系建立

### 方案选择

| 方案 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| 绝对方位 | 建筑/室内 | 不随视角改变，绝对清晰 | 需要先确定朝向 |
| 相对方位 | 物体关系 | 直观，易于理解 | 随视角改变，容易混淆 |
| 混合方案（推荐） | 通用 | 结合两者优势 | — |

**混合方案**：用绝对方位描述墙面和大结构，用相对方位描述物体间关系。
例："东墙（画面背景）的书桌左侧（北侧）"

### 建立流程

1. **确定场景的绝对朝向**
   "这个房间的门朝南，窗户朝东"

2. **确定摄影机位置和朝向**
   "摄影机在西墙，面向东墙"

3. **标注四个方向对应的画面位置**
   - 东墙 = 画面背景（正前方）
   - 西墙 = 摄影机位置（身后，不可见）
   - 北墙 = 画面左侧
   - 南墙 = 画面右侧

4. **用这个系统描述所有元素**

## 可见性判断算法

### 规则1：摄影机后方不可见
```
if 物体在摄影机所在墙面:
    物体不可见

例子: 摄影机在西墙 → 西墙上的书柜不可见
```

### 规则2：遮挡关系
```
if 物体A在物体B和摄影机之间:
    物体B被部分/完全遮挡

例子: 椅子在书桌和摄影机之间 → 椅子遮挡部分书桌
```

### 规则3：视角范围
```
if 物体在摄影机视角范围外:
    物体不可见

例子: 广角镜头(24mm) → 视角约84°
      标准镜头(50mm) → 视角约47°
```

### 规则4：距离与细节
```
if 物体距离摄影机很远:
    物体可见但细节模糊

例子: 远处的海图 → 可见但文字不清晰
```

### 判断流程
对于每个物体：
1. 确定物体位置（哪面墙/哪个区域）
2. 确定摄影机位置和朝向
3. 应用规则 1-4
4. 得出结论：可见/不可见/部分可见
5. 如果可见，确定在画面中的位置

## 空间关系描述模板

### 模板1：机位描述
```
Camera on [墙面/位置], shooting [方向], [视角类型].
Eye level / high angle / low angle.
摄影机在[位置], 面向[方向], [视角描述]。
```

### 模板2：空间关系声明
```
[方位1](摄影机位置, 身后) = 看不到
[方位2](正前方) = 画面背景中央
[方位3](左手边) = 画面左侧
[方位4](右手边) = 画面右侧
```

### 模板3：物体位置描述
```
[物体名称]:
- 位于[绝对位置](哪面墙/哪个区域)
- 在画面中的位置: [相对位置](前景/中景/背景, 左/中/右)
- 与[参照物]的关系: [方位+距离]
```

### 模板4：多物体关系描述
```
[物体A] 在 [物体B] 的 [方位], 距离约 [数值]。
从[视角]看, [物体A] [遮挡/并列/前后] [物体B]。
```

## 空间错误预防清单

| 错误类型 | 预防措施 | 检查项 |
|----------|----------|--------|
| 显示摄影机后方物体 | 提示词开头明确摄影机位置；负向提示词排除 | □ 摄影机在X墙 → X墙物体不应出现 |
| 左右方位混淆 | 优先使用绝对方位；左右描述注明视角 | □ 所有方位描述都有明确参照系 |
| 距离关系不合理 | 使用具体数值；参考真实建筑尺寸 | □ 所有距离都有数值 |
| 遮挡关系错误 | 明确前后层次；描述"A在B前方" | □ 前景/中景/背景清晰划分 |
| 视角与内容不匹配 | 广角→更多空间；长焦→压缩空间 | □ 镜头类型与场景规模匹配 |

## 经典错误案例

**案例：书柜错误**
- 错误：西向东拍时显示了西墙书柜
- 根因：未理解"摄影机在西墙"的含义
- 教训：摄影机所在墙面永远不可见
- 预防：开头明确空间关系，负向提示词排除

**案例：门光错误**
- 错误：门窗纸发光，而非门缝发光
- 根因：未明确光源的精确位置
- 教训：光源描述要精确到具体部位
- 预防：使用"only X emits light"句式
