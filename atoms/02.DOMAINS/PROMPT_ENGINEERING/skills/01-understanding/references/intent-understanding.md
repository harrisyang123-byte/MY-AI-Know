# Skill 1.2: 需求意图理解

## 核心思想

用户说的 ≠ 用户想要的。需要透过表面需求，理解深层意图。

## 需求金字塔

```
Level 4: 深层动机 (Ultimate goal)
  用户的最终目的
  例: "我在创作一个历史题材的视觉小说"

Level 3: 情感需求 (Why user wants it)
  用户想要的感觉和氛围
  例: "我想要营造神秘、紧张的历史氛围"

Level 2: 功能需求 (What user wants)
  用户想要达成的具体目标
  例: "我需要从西墙视角看整个房间"

Level 1: 表面需求 (What user says)
  用户直接说出的内容
  例: "生成一个西向东的场景"
```

分析方法：
1. 从 Level 1 开始，逐层深挖
2. 通过提问或推断，理解更深层次
3. 在不确定时，询问而非假设
4. 记录用户的深层动机，指导后续决策

## 需求冲突识别

### 类型1：技术冲突
- 例子："要超高清，但要快速生成"
- 分析：高清需要更多时间
- 解决：询问优先级（质量 vs 速度）

### 类型2：风格冲突
- 例子："要写实，但要有艺术感"
- 分析：写实和艺术化有时矛盾
- 解决：明确"艺术感"的具体含义

### 类型3：内容冲突
- 例子："要昏暗氛围，但要看清所有细节"
- 分析：昏暗会牺牲细节可见度
- 解决：平衡光线，或生成多个版本

### 类型4：逻辑冲突
- 例子："从西向东拍，要看到西墙的书柜"
- 分析：空间关系不可能
- 解决：指出冲突，提供替代方案

### 处理流程
1. 识别冲突
2. 分析根源（用户误解？表达不清？）
3. 温和地指出冲突
4. 提供 2-3 个解决方案
5. 让用户选择

## 需求补全策略

### 补全原则
1. 关键信息必须询问
2. 次要信息可以假设（但要说明）
3. 基于历史偏好做推断
4. 提供默认值，允许用户修改

### 图像生成关键信息清单

必须明确：
- [ ] 主题/内容（生成什么）
- [ ] 视角/构图（从哪里看）
- [ ] 风格/氛围（什么感觉）

可以假设：
- [ ] 尺寸（默认常用比例）
- [ ] 色彩（根据主题推断）
- [ ] 细节程度（根据用途推断）

可以询问：
- [ ] 参考图（有没有喜欢的例子）
- [ ] 特殊要求（有没有特别注意的）

### 血全示例

用户说："生成一个办公室"

方案 A：直接询问
- "您想要什么风格的办公室？现代/传统/科技感？"
- "需要包含哪些元素？桌子/电脑/书柜？"

方案 B：提供默认值
- "我将生成一个现代简约风格的办公室，平视角度，包含办公桌、电脑、书柜。如果需要调整，请告诉我。"

方案 C：提供选项（推荐）
- "请选择办公室类型：
  1. 现代科技公司（开放式、简约、明亮）
  2. 传统律师事务所（木质、庄重、经典）
  3. 创意工作室（个性化、色彩丰富、随性）"

**推荐方案 C**：给用户选择，但不需要从零描述。
