# Level 1: 理解能力

理解 = 信息提取 + 意图推断 + 空间建模 + 逻辑验证

## 三个核心技能

| 技能 | 核心问题 | 产出 |
|------|----------|------|
| 1.1 多模态分析 | 输入里有什么？ | 关键词列表、风格标签、技术参数 |
| 1.2 意图理解 | 用户到底要什么？ | 补全后的需求描述、待确认问题 |
| 1.3 空间推理 | 空间关系是什么？ | 坐标系、可见性判断、空间描述 |

## 协同工作流

```
用户输入
 │
 ▼
Skill 1.1 多模态分析
 ├── 文本：提取关键词、约束、技术参数
 ├── 图片：分析风格、元素、光线、构图
 └── 输出：关键词列表 + 风格标签 + 参考描述
 │
 ▼
Skill 1.2 意图理解
 ├── 显性需求 → 隐性需求 → 深层动机
 ├── 冲突识别 + 需求补全
 └── 输出：补全需求 + 待确认问题
 │
 ▼
Skill 1.3 空间推理
 ├── 建立坐标系 + 判断可见性 + 描述空间关系
 └── 输出：空间描述 + 可见/不可见元素列表
 │
 ▼
整合输出 → 传递给 Level 2 规划
```

## 理解阶段检查清单

```
□ 多模态分析
  □ 如果有图片，是否已分析？
  □ 技术参数是否提取？
  □ 风格标签是否明确？

□ 需求理解
  □ 显性需求是否全部识别？
  □ 隐性需求是否合理推断？
  □ 需求冲突是否识别？
  □ 缺失信息是否补全或询问？

□ 空间推理
  □ 坐标系是否建立？
  □ 摄影机位置是否明确？
  □ 可见性是否正确判断？

□ 整体检查
  □ 是否有自相矛盾的地方？
  □ 是否有模糊不清的描述？
  □ 是否可以直接用于生成？
```
