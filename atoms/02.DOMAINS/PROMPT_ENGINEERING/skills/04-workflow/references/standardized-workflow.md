# Skill 4.1: 标准化工作流

## 三级流程模板

### 流程模板1：简单图像生成（5-8分钟）

```
Phase 1: 快速理解 (1-2分钟)
  □ 用户要什么？
  □ 有参考图吗？
  □ 风格要求？
  □ 尺寸要求？

Phase 2: 工具选择 (30秒)
  □ 用户指定工具？→ 使用指定工具
  □ 需要文字？→ nano_banana_2
  □ 需要艺术化？→ midjourney
  □ 默认 → gpt_image_2

Phase 3: 提示词构建 (2-3分钟)
  □ 核心概念(1句话)
  □ 关键元素(列举)
  □ 风格描述(参考或关键词)
  □ 技术参数(如果需要)

Phase 4: 参数配置 (30秒)
  □ size/aspect_ratio: 根据用途
  □ quality/resolution: 默认 medium/1K
  □ task_type: 根据是否有参考图

Phase 5: 执行生成

Phase 6: 快速验证 (1分钟)
  □ 核心概念对吗？
  □ 关键元素都有吗？
  □ 明显错误吗？
    ├─ 是 → 快速修正，重新生成
    └─ 否 → 交付
```

### 流程模板2：中等复杂度图像生成（20-40分钟）

```
Phase 1: 深度理解 (3-5分钟)
  □ 分析参考图(如果有)
  □ 识别关键需求
  □ 识别潜在难点
  □ 确认不确定的地方

Phase 2: 策略规划 (2-3分钟)
  □ 选择工具和方法
  □ 决定是否需要分步
  □ 预判可能的问题
  □ 准备备选方案

Phase 3: 详细提示词 (5-8分钟)
  □ 按8段式结构构建
  □ 精确化描述
  □ 空间关系验证
  □ 添加负向提示词

Phase 4: 首次生成

Phase 5: 详细验证 (3-5分钟)
  □ 需求符合度检查
  □ 空间关系检查
  □ 细节准确性检查
  □ 识别问题并诊断

问题严重吗？
├─ 是 → Phase 6: 迭代优化 (5-10分钟)
│        □ 修正提示词
│        □ 重新生成
│        □ 再次验证
└─ 否 → 交付
```

### 流程模板3：复杂场景/系列作品（1-3小时）

```
Phase 1: 全面分析 (10-15分钟)
  □ 深度分析所有参考材料
  □ 理解整体项目目标
  □ 识别一致性要求
  □ 列出所有子任务
  □ 识别依赖关系

Phase 2: 整体规划 (10-15分钟)
  □ 制定执行顺序
  □ 建立风格基准
  □ 准备可复用资产
  □ 设计一致性检查点

Phase 3: 建立基准 (15-20分钟)
  □ 生成第一个基准作品
  □ 详细验证
  □ 与用户确认方向
  □ 提取可复用元素
  □ 建立提示词模板

Phase 4: 批量生成
  □ 基于模板生成其他作品
  □ 保持一致性
  □ 逐个验证

Phase 5: 统一调整 (10-20分钟)
  □ 对比所有作品
  □ 识别不一致之处
  □ 统一调整
  □ 最终验证

Phase 6: 交付与文档 (5-10分钟)
  □ 整理所有输出
  □ 记录提示词和参数
  □ 总结经验教训
  □ 建立可复用资产库
```

## 任务复杂度评估

| 评估维度 | 简单 | 中等 | 复杂 |
|----------|------|------|------|
| 元素数量 | 1-2个 | 3-5个 | 6+个 |
| 空间关系 | 无/简单 | 有但清晰 | 复杂 |
| 参考图 | 无 | 1-2张 | 3+张 |
| 特殊要求 | 无 | 有 | 多个 |
| 系列作品 | 否 | 否 | 是 |

根据评估结果选择对应流程模板。

## 检查清单体系

### 执行前检查

```
□ 理解检查
  □ 我完全理解用户要什么了吗？
  □ 有不确定的地方吗？(如果有，先问清楚)
  □ 参考图都分析了吗？

□ 工具检查
  □ 用户指定工具了吗？
  □ 我选择的工具合适吗？
  □ 有备选方案吗？

□ 提示词检查
  □ 核心概念清晰吗？
  □ 关键元素都包含了吗？
  □ 空间关系正确吗？
  □ 有自相矛盾的描述吗？
  □ 负向提示词准备了吗？

□ 风险检查
  □ 可能出现什么问题？
  □ 如何预防？
```

### 执行后检查

```
□ 需求符合度 — 是用户要的东西吗？关键元素都有吗？
□ 空间关系 — 视角正确吗？不该出现的元素出现了吗？
□ 视觉质量 — 清晰度足够吗？光影合理吗？
□ 风格一致性 — 与参考图一致吗？与系列其他作品一致吗？
□ 细节准确性 — 历史/文化细节对吗？文字正确吗？
□ 交付准备 — 需要进一步优化吗？用户会满意吗？
```

### 迭代检查

```
□ 问题识别 — 问题是什么？严重程度？根本原因？
□ 解决方案 — 如何修复？需要改什么？只改一个维度吗？
□ 执行验证 — 改动是否有效？有没有引入新问题？
□ 迭代控制 — 这是第几次迭代？是否超过3次？是否需要换策略？
```

## 模板库建设

### 提示词模板示例

**清代室内场景模板**：
```
【核心概念】{年代}年{季节}{时间}，{场景类型}
【视角】摄影机在{位置}，面向{方向}，{景别}
【空间】房间{尺寸}，层高{高度}
  {方位1}=背景，{方位2}=前景，{方位3}=左侧，{方位4}=右侧
【背景层】- {墙面}: {窗户/装饰描述}
【中景层】- {主要家具}: {详细描述}
  - {人物}(如果有): {外观和动作}
【前景层】- {前景元素}: {详细描述}
【光线】主光: {光源类型}({色温})，{强度描述}
  辅光: {光源类型}({色温})，{强度描述}
【风格】{艺术风格参考}，{渲染方式}，{质感要求}
```

**产品摄影模板**：
```
【产品】{产品名称}，{材质}，{颜色}，{尺寸}
【视角】{角度}(45度俯视/正面平视/侧面特写)
【背景】{背景类型}(纯色/渐变/场景/透明)，{背景描述}
【光线】主光: {位置和类型}，辅光: {位置和类型}
【构图】{产品位置}(居中/三分法)，{留白}
【风格】{风格类型}(简约现代/奢华/自然)
【技术】{渲染要求}(photorealistic, studio lighting)
```

### 使用方法

1. 识别任务类型
2. 查找对应模板
3. 复制并定制
4. 执行
5. 如果效果好，更新模板
6. 如果发现新模式，创建新模板
