# Skill 4.2: 知识积累与复用

## 五库经验体系

### 1. 成功案例库

记录内容：
- 任务描述
- 最终提示词
- 参数配置
- 生成结果
- 成功关键因素
- 可复用元素

示例条目：
```
任务: 清代海军衙门办公室，西向东视角
提示词: [完整的西向东拍提示词]
参数: nano_banana_2, REFERENCE_TO_IMAGE, 16:9, 1K
成功因素:
  - 明确的空间坐标系建立
  - 详细的负向提示词
  - 结构化的层次描述
可复用:
  - 清代室内空间描述框架
  - 烛光照明描述方法
  - 西向东视角空间处理
```

### 2. 失败案例库

记录内容：
- 任务描述
- 尝试的方法
- 失败表现
- 失败原因
- 教训
- 正确做法

示例条目：
```
任务: 清代办公室西向东
尝试: 首次生成包含了西墙书柜描述
失败表现: 书柜出现在画面中(不应该)
失败原因: 没有理解"摄影机在西墙"意味着西墙不可见
教训:
  - 必须明确摄影机位置
  - 必须排除不可见元素
  - 使用负向提示词预防
正确做法:
  - 建立空间坐标系
  - 列出不可见区域
  - 添加负向提示词
```

### 3. 技巧库

记录内容：
- 技巧名称
- 适用场景
- 具体方法
- 示例
- 效果

示例条目：
```
技巧: 使用色温值精确控制光线
适用: 需要特定氛围的照明
方法:
  - 不说"温暖的光"，说"2800K琥珀色烛光"
  - 不说"冷光"，说"6000K冷白雪光"
示例: "主光源:单支蜡烛(2800K)，辅光:月光(6000K)"
效果: 光线色调更精确，氛围更可控
```

### 4. 工具特性库

记录内容：
- 工具名称
- 擅长/不擅长的任务
- 参数最佳实践
- 常见问题及解决

示例条目：
```
工具: generate_image_nano_banana_2
擅长: 文字渲染、参考图驱动生成、多分辨率支持
不擅长: 极度抽象的艺术风格、超复杂空间关系
参数实践:
  - 文字任务用2K分辨率
  - EDIT模式必须匹配原图比例
  - task_type要准确选择
常见问题:
  - 比例不匹配导致变形 → EDIT时aspect_ratio匹配原图
```

### 5. 领域知识库

记录内容：
- 领域名称
- 关键特征
- 常用描述词汇
- 参考资料
- 注意事项

示例条目：
```
领域: 清代官方建筑室内
关键特征:
  - 深色硬木家具
  - 红色立柱
  - 白墙+深色护墙板
  - 格栅窗
  - 书法/地图装饰
常用词汇:
  - "深色硬木"、"红漆立柱"、"白色粉刷墙面"
  - "深色木质护墙板"、"格栅窗"、"窗棂"
注意事项:
  - 避免现代元素
  - 注意时代一致性
  - 光源以蜡烛/油灯为主
```

## 知识库使用流程

```
遇到新任务 → 查询经验库
├─ 有类似成功案例？→ 复用提示词和方法
├─ 有类似失败案例？→ 避免同样错误
├─ 有相关技巧？→ 应用技巧
├─ 有工具经验？→ 参考最佳实践
└─ 有领域知识？→ 使用准确描述

完成任务 → 更新经验库
├─ 成功了？→ 添加到成功案例库
├─ 失败了？→ 添加到失败案例库
├─ 发现新技巧？→ 添加到技巧库
├─ 学到工具特性？→ 更新工具特性库
└─ 积累领域知识？→ 更新领域知识库
```

## 个人风格建设

### 五个维度

1. **工具偏好** — 最常用的工具、使用场景、参数习惯
2. **提示词风格** — 偏好的结构、描述模式、语言习惯
3. **执行习惯** — 偏好的工作流程、检查方法、时间分配
4. **擅长领域** — 最擅长的任务类型、积累最多的领域
5. **改进方向** — 当前的弱项、想要提升的能力、学习计划

### 建设方法

1. **定期回顾**（每周/每月）— 回顾所有任务，识别模式和偏好
2. **有意识地实践** — 在安全任务中尝试新方法，对比效果
3. **记录和反思** — 记录每次选择和原因，反思为什么这样选择
4. **持续优化** — 保留有效习惯，改进低效部分
5. **形成体系** — 将零散经验系统化，建立可传授的知识
