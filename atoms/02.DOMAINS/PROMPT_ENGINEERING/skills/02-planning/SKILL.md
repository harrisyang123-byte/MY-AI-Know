# Level 2: 规划能力

规划 = 分解 + 排序 + 选择 + 监控 + 调整

## 三个核心技能

| 技能 | 核心问题 | 产出 |
|------|----------|------|
| 2.1 任务分解与排序 | 怎么拆？先做什么？ | 任务树 + 依赖图 + 执行序列 |
| 2.2 策略选择与工具匹配 | 用什么方法？什么工具？ | 策略模式 + 工具选择 + 参数配置 |
| 2.3 动态调整与应急预案 | 出问题怎么办？ | 监控框架 + 应急预案 + 调整决策 |

## 协同工作流

```
Level 1 理解输出
 │
 ▼
Skill 2.1 任务分解与排序
 ├── 自顶向下/自底向上/里程碑法
 ├── 标注依赖关系（强依赖/弱依赖/无依赖/互斥）
 └── 输出：任务树 + 执行序列
 │
 ▼
Skill 2.2 策略选择与工具匹配
 ├── 决策树：任务类型 → 工具选择 → 参数配置
 ├── 策略模式：渐进式/参考驱动/搜索增强/分治法
 └── 输出：工具 + 参数 + 策略模式
 │
 ▼
Skill 2.3 动态调整与应急预案
 ├── 监控维度：进度/质量/资源
 ├── 应急预案：工具失败/理解偏差/需求变更/循环/时间压力
 └── 输出：监控检查点 + 应急预案
 │
 ▼
整合输出 → 传递给 Level 3 执行
```

## 规划核心原则

1. **计划要详细，但不要僵化** — 详细提供方向，随时准备调整
2. **优先级大于顺序** — 先做最重要的事，而非最简单的事
3. **并行优于串行** — 能同时做的就不要排队，但注意依赖关系
4. **预案优于临时应对** — 提前想好"如果 X 失败了怎么办"
5. **沟通优于猜测** — 不确定时，问用户，而非自己假设
