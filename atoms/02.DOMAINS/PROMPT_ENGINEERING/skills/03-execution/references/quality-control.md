# Skill 3.3: 质量控制与验证

## 6维度验证清单

### 维度1：需求符合度 ★★★★★（最重要）

- [ ] 核心概念是否正确？生成的是用户要的东西吗？
- [ ] 关键元素是否齐全？用户要求的元素都有吗？
- [ ] 用户的特殊要求是否满足？

**不符合 → 立即重新生成，不要交付**

### 维度2：空间关系正确性 ★★★★☆

- [ ] 视角是否正确？摄影机位置和朝向对吗？
- [ ] 元素位置是否合理？前后、左右、距离关系？
- [ ] 不应该出现的元素是否出现了？

**错误 → 修正提示词，重新生成**

### 维度3：视觉质量 ★★★★☆

- [ ] 分辨率是否足够？是否模糊？
- [ ] 细节是否清晰？重要元素是否清晰可辨？
- [ ] 光影是否自然？光源位置和阴影方向？

**质量不足 → 提高 quality/resolution 参数**

### 维度4：风格一致性 ★★★☆☆

- [ ] 与参考图风格是否一致？
- [ ] 与系列其他图是否一致？
- [ ] 风格是否符合预期（写实 vs 艺术化）？

**不一致 → 调整提示词或参考图**

### 维度5：细节准确性 ★★★☆☆

- [ ] 历史/文化细节是否准确？
- [ ] 技术细节是否合理？比例关系？
- [ ] 文字是否正确？

**错误 → 根据严重程度决定是否重新生成**

### 维度6：整体美感 ★★☆☆☆

- [ ] 构图是否美观？有视觉焦点吗？
- [ ] 色彩是否和谐？对比度合适吗？
- [ ] 氛围是否到位？

**不满意 → 可以尝试优化，但不是必须**

## 验证流程

```
Step 1: 快速扫描 (5秒)
  第一眼感觉对吗？有明显错误吗？
  → 不对 → 直接重新生成

Step 2: 需求对照 (30秒)
  对照用户需求逐项检查，核心要素都有吗？
  → 缺少关键元素 → 重新生成

Step 3: 细节检查 (1-2分钟)
  放大查看重要区域，检查空间关系和光影逻辑
  → 有明显错误 → 决定修正还是重新生成

Step 4: 整体评估 (30秒)
  如果是你，会满意吗？
  → 不满意 → 分析原因，调整策略

Step 5: 用户视角 (1分钟)
  站在用户角度看，用户会满意吗？
  → 不确定 → 可以询问用户
```

## 问题决策矩阵

| 问题严重度 | 修复成本 | 决策 |
|-----------|---------|------|
| 严重 | 低 | 立即修复 |
| 严重 | 高 | 重新生成 |
| 中等 | 低 | 修复 |
| 中等 | 高 | 评估后决定 |
| 轻微 | 低 | 可选修复 |
| 轻微 | 高 | 接受现状 |

## 6类常见问题诊断

### 问题1：空间关系错误

| 根因 | 解决方案 |
|------|----------|
| 提示词中空间描述不清晰 | 明确摄影机位置，使用绝对方位 |
| 提示词中包含了不应该出现的元素 | 删除该元素，添加到负向提示词 |
| 工具对空间关系理解有限 | 更详细的空间描述，或换工具 |

### 问题2：风格不符

| 根因 | 解决方案 |
|------|----------|
| 风格描述词不够强 | 增加风格关键词的权重和数量 |
| 工具特性不匹配 | 艺术化→Midjourney，写实→GPT-Image-2 |
| 参考图没有正确使用 | 在提示词中明确说明参考图的作用 |

### 问题3：元素缺失或错误

| 根因 | 解决方案 |
|------|----------|
| 提示词中没有提到该元素 | 添加该元素的详细描述 |
| 提示词过于复杂，AI 忽略了某些元素 | 简化提示词，或分步生成 |
| 元素描述不够具体 | 增加细节描述，使用数值 |

### 问题4：光影不自然

| 根因 | 解决方案 |
|------|----------|
| 光源描述不清晰 | 明确光源类型、位置、强度、色温 |
| 多光源描述混乱 | 分主光、辅光、环境光分别描述 |
| 工具自动调整了光线 | 在提示词中强调"精确遵循光源设置" |

### 问题5：质量问题

| 根因 | 解决方案 |
|------|----------|
| 分辨率设置太低 | 提高 resolution 参数 |
| quality 设置太低 | 提高 quality 参数 |
| 场景过于复杂 | 简化场景，或分步生成后合成 |

### 问题6：文字渲染错误

| 根因 | 解决方案 |
|------|----------|
| 工具不擅长文字渲染 | 使用 nano_banana_2 |
| 文字描述不够明确 | 用引号标注文字内容，说明字体和位置 |
| 文字太小或太复杂 | 增大文字，简化内容，提高分辨率 |

## 迭代优化策略

### 四种迭代模式

| 模式 | 适用场景 | 方法 |
|------|----------|------|
| 渐进式 | 方向正确，需要微调 | V1 → 反馈 → V2 → 反馈 → V3 |
| 探索式 | 方向不明 | V1a, V1b, V1c → 用户选择 → V2 |
| 分步式 | 超复杂场景 | 背景 → 中景 → 前景 → 整合 |
| 对比式 | 不确定改动效果 | 生成 V1 和 V2，用户对比选择 |

### 迭代记录模板

```
Iteration N:
  改动: [这次改了什么]
  原因: [为什么要这样改]
  结果: [生成结果如何]
  用户反馈: [用户怎么说]
  下一步: [下次要改什么]
```
