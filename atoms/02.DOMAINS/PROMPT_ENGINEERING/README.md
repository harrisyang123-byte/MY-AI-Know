# 02.DOMAINS/PROMPT_ENGINEERING — 提示词工程领域

基于 AI 图像生成实战经验提炼的提示词工程方法论。五层能力体系，从理解需求到持续进化，覆盖提示词创作全链路。

## 核心理念

提示词是与 AI 工具之间的**合同**——写得越精确，执行结果越可控。好的提示词不是天赋，而是可训练、可复用、可进化的系统工程。

## 五层能力体系

```
Level 5: 持续进化 ── 元认知 + 主动学习
Level 4: 建立流程 ── 标准化 + 知识沉淀
Level 3: 执行能力 ── 提示词工程 + 工具精通 + 质量控制
Level 2: 规划能力 ── 任务分解 + 策略选择 + 动态调整
Level 1: 理解能力 ── 多模态分析 + 意图理解 + 空间推理
```

| 层级 | 核心技能 | 关键产出 |
|------|----------|----------|
| L1 理解 | 多模态分析、意图理解、空间推理 | 需求分析报告、空间坐标系 |
| L2 规划 | 任务分解、策略选择、动态调整 | 执行计划、工具选择、应急预案 |
| L3 执行 | 提示词工程、工具精通、质量控制 | 可执行提示词、生成图像 |
| L4 流程 | 标准化工作流、知识积累 | 流程模板、检查清单、经验库 |
| L5 进化 | 元认知、主动学习 | 能力雷达图、决策树优化 |

## 标准工作流

```
用户需求
 │
 ▼
Level 1: 理解
 ├── 多模态分析：提取关键词、风格标签、技术参数
 ├── 意图理解：显性需求 → 隐性需求 → 深层动机
 └── 空间推理：建立坐标系、判断可见性、描述空间关系
 │
 ▼
Level 2: 规划
 ├── 任务分解：自顶向下拆解，标注依赖关系
 ├── 策略选择：工具匹配、参数配置、组合模式
 └── 应急预案：工具失败、理解偏差、需求变更
 │
 ▼
Level 3: 执行
 ├── 提示词构建：8段式结构 → 精确化 → 空间验证 → 负向提示词
 ├── 工具调用：参数调优、工具组合
 └── 质量验证：6维度检查清单
 │
 ▼
Level 4: 流程化
 ├── 模板提取：提示词模板、参数模板、工作流模板
 └── 知识沉淀：成功/失败案例库、技巧库、领域知识库
 │
 ▼
Level 5: 进化
 ├── 元认知反思：执行日志、能力雷达图、决策树优化
 └── 主动学习：工具探索、技能拓展、领域深耕、创新实验
```

## Skills 说明

| Skill | 说明 | 触发关键词 |
|-------|------|-----------|
| 01-understanding | 多模态分析 + 意图理解 + 空间推理 | "分析需求"、"理解意图"、"空间关系" |
| 02-planning | 任务分解 + 策略选择 + 动态调整 | "规划任务"、"选择工具"、"应急预案" |
| 03-execution | 提示词构建 + 工具精通 + 质量控制 | "生成提示词"、"调参"、"验证质量" |
| 04-workflow | 标准化流程 + 知识积累 | "建立流程"、"模板"、"知识库" |
| 05-evolution | 元认知 + 主动学习 | "反思"、"改进"、"学习" |

## Agent

**prompt-architect** — 提示词架构师
集成五层能力的全能型 Agent，从需求理解到提示词交付，支持单图生成、系列作品、复杂迭代等场景。

## 灵活入口

| 场景 | 操作 |
|------|------|
| 从头开始新项目 | 加载 `prompt-architect`，走完整五层流程 |
| 只需要生成提示词 | 跳过 L1-L2，直接进入 L3 执行 |
| 需要修正已有提示词 | 加载 L3 质量控制 + L5 元认知 |
| 建立团队标准 | 加载 L4 流程化 |
| 提升个人能力 | 加载 L5 进化 |
